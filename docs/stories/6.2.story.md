## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** criar e gerenciar quizzes com questões para cada aula pelo painel admin,
**so that** os alunos possam ser avaliados e eu possa controlar o aproveitamento de cada conteúdo.

## Business Value

Sem quiz builder, Fábio não consegue avaliar alunos nem condicionar avanço de aula. O quiz é central no modelo pedagógico da plataforma — é o que diferencia um curso sério de uma simples playlist de vídeos.

## Complexity Estimate

**T-shirt:** L (Large) — ~12h
**Story Points:** 5

## Scope

### IN

- Página `/admin/cursos/[courseId]/aulas/[lessonId]/quiz` — editor de quiz de uma aula
- Ou, **preferencialmente**, modal/panel inline dentro do editor de aulas (Story 6.1) — botão "Gerenciar Quiz" abre o Quiz Builder
- **CRUD de Quiz** (1 quiz por aula):
  - Criar quiz (título, % aprovação, tentativas permitidas)
  - Editar quiz
  - Deletar quiz (apaga todas as questões)
- **CRUD de Questões** dentro do quiz:
  - Tipos: `multiple_choice` (4 opções, 1 correta), `true_false`, `open` (dissertativa)
  - Criar questão: body (enunciado), tipo, opções (para múltipla escolha), resposta correta
  - Editar questão
  - Deletar questão
  - Reordenação de questões (↑ ↓)
- **APIs admin:**
  - `POST /api/admin/lessons/[lessonId]/quiz` — criar quiz
  - `PATCH /api/admin/quizzes/[id]` — atualizar quiz
  - `DELETE /api/admin/quizzes/[id]` — deletar quiz + questões
  - `POST /api/admin/quizzes/[quizId]/questions` — adicionar questão
  - `PATCH /api/admin/questions/[id]` — atualizar questão
  - `DELETE /api/admin/questions/[id]` — deletar questão
- Todas as APIs verificam `role = 'admin'`
- Feedback: loading, sucesso, erro

### OUT

- Preview de como o quiz aparece para o aluno
- Importar questões de CSV/banco externo
- Questões com imagem
- Estatísticas de aproveitamento (relatórios — Epic 10)
- Lógica de bloqueio de próxima aula (implementada no player — Story 8.2)

## Dependencies

- Story 6.1 Done: aulas existem e são exibidas no admin
- Tabelas `quizzes` e `questions` já no schema (migration 5)
- Tipo enum `question_type` já no banco: `multiple_choice`, `true_false`, `open`

## Acceptance Criteria

- **AC-1:** Na edição de uma aula, existe botão "Quiz" que abre o Quiz Builder (modal ou painel inline)
- **AC-2:** Se a aula não tem quiz, exibe botão "Criar quiz" com form: título, % aprovação (padrão 70), tentativas (padrão 3)
- **AC-3:** Se a aula já tem quiz, exibe as questões cadastradas e botão "+ Adicionar questão"
- **AC-4:** Form de questão múltipla escolha: enunciado + 4 campos de opção (A, B, C, D) + select "Resposta correta"
- **AC-5:** Form de questão V/F: enunciado + radio Verdadeiro/Falso como resposta correta
- **AC-6:** Form de questão dissertativa (`open`): apenas enunciado (sem resposta correta — correção manual)
- **AC-7:** Editar/deletar questão existente funciona sem reload
- **AC-8:** Todas as 6 APIs retornam 401/403 para não-admin
- **AC-9:** `npm run lint` e `npm run typecheck` sem erros

## Technical Notes

### Estrutura das questões no banco

```typescript
// multiple_choice
{
  type: 'multiple_choice',
  body: 'Qual equipamento usa ultrassom micro e macrofocado?',
  options: ['HIFU', 'Radiofrequência', 'Eletrocarbo', 'Criolipólise'],
  correct_answer: 'HIFU'
}

// true_false
{
  type: 'true_false',
  body: 'A criolipólise pode ser aplicada em todas as regiões do corpo.',
  options: [],
  correct_answer: 'false'
}

// open
{
  type: 'open',
  body: 'Descreva o protocolo de aplicação de radiofrequência para flacidez.',
  options: [],
  correct_answer: null
}
```
