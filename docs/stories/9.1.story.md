## Story

**As a** visitante / motor de busca / administrador da plataforma,
**I want** que as páginas públicas sejam indexáveis, tenham Open Graph correto e a aplicação tenha headers de segurança,
**so that** o SEO seja eficiente, os links compartilhados nas redes sociais mostrem preview correto, e a plataforma passe em auditorias básicas de segurança e Lighthouse antes do go-live.

## Business Value

Sem sitemap.xml e robots.txt, o Google não indexa o catálogo de cursos — zero tráfego orgânico. Sem OG tags, links compartilhados no WhatsApp/Instagram aparecem sem preview. Sem headers de segurança (CSP, HSTS), a plataforma reprova em audits de segurança e Lighthouse. Esta story fecha o Sprint 7 (QA Go-Live do PRD seção 7) antes do deploy em produção.

## Technical Notes

### App Router Sitemap (src/app/sitemap.ts)

```typescript
import type { MetadataRoute } from 'next'
import { createClient } from '@/lib/supabase/server'

const APP_URL = process.env.NEXT_PUBLIC_APP_URL ?? 'https://ead.fabioborgesoficial.com.br'

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  const supabase = createClient()
  const { data } = await supabase.from('courses').select('slug, updated_at').eq('published', true)

  const courseUrls: MetadataRoute.Sitemap = (data ?? []).map((c) => ({
    url: `${APP_URL}/cursos/${(c as { slug: string }).slug}`,
    lastModified: (c as { updated_at?: string }).updated_at ?? new Date().toISOString(),
    changeFrequency: 'weekly',
    priority: 0.8,
  }))

  return [
    { url: APP_URL, lastModified: new Date(), changeFrequency: 'monthly', priority: 1.0 },
    {
      url: `${APP_URL}/cursos`,
      lastModified: new Date(),
      changeFrequency: 'weekly',
      priority: 0.9,
    },
    {
      url: `${APP_URL}/planos`,
      lastModified: new Date(),
      changeFrequency: 'monthly',
      priority: 0.7,
    },
    ...courseUrls,
  ]
}
```

[Source: Next.js docs — App Router `src/app/sitemap.ts` automaticamente serve `/sitemap.xml`]

### App Router Robots (src/app/robots.ts)

```typescript
import type { MetadataRoute } from 'next'

const APP_URL = process.env.NEXT_PUBLIC_APP_URL ?? 'https://ead.fabioborgesoficial.com.br'

export default function robots(): MetadataRoute.Robots {
  return {
    rules: { userAgent: '*', allow: '/' },
    sitemap: `${APP_URL}/sitemap.xml`,
  }
}
```

### OG Tags — padrão para generateMetadata existente

Em `/cursos/[slug]`, `generateMetadata` já existe. Adicionar campo `openGraph`:

```typescript
return {
  title: `${course.title} | Mentoria Fábio Borges`,
  description: course.description ?? `Conheça o curso ${course.title}...`,
  openGraph: {
    title: `${course.title} | Mentoria Fábio Borges`,
    description: course.description ?? `Conheça o curso ${course.title}...`,
    url: `${APP_URL}/cursos/${course.slug}`,
    type: 'website',
    images: course.thumbnail_url
      ? [{ url: course.thumbnail_url, width: 1200, height: 630 }]
      : [{ url: `${APP_URL}/og-default.png`, width: 1200, height: 630 }],
  },
}
```

Para `/cursos` e `/planos` (metadata estático), adicionar `openGraph` ao objeto `metadata` existente.

> **Nota:** `og-default.png` não precisa existir agora — OG image fallback genérica é aceita mesmo sem arquivo. Pode ser criado depois com identidade visual. O campo `images` no metadata não quebra se a imagem não existir.

### Security Headers em next.config.mjs

```javascript
const nextConfig = {
  async headers() {
    return [
      {
        source: '/(.*)',
        headers: [
          { key: 'X-Frame-Options', value: 'SAMEORIGIN' },
          { key: 'X-Content-Type-Options', value: 'nosniff' },
          { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },
          { key: 'Permissions-Policy', value: 'camera=(), microphone=(), geolocation=()' },
          {
            key: 'Strict-Transport-Security',
            value: 'max-age=63072000; includeSubDomains; preload',
          },
        ],
      },
    ]
  },
}
```
