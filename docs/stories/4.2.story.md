## Story

**As a** aluno que concluiu um curso com aproveitamento,
**I want** receber e baixar um certificado de conclusão com QR Code verificável,
**so that** possa comprovar minha formação e compartilhar minha conquista profissionalmente.

## Business Value

O certificado é o produto percebido de maior valor para o aluno — é a prova tangível do investimento na mentoria. Sua ausência é um bloqueador direto de vendas: alunos e alunas do nicho de estética precisam comprovar formação para clientes e estabelecimentos. Certificados com QR Code verificável diferencia a plataforma de concorrentes com PDFs simples. Esta story também habilita o contador de certificados já exibido no dashboard e perfil do aluno.

## Complexity Estimate

**T-shirt:** M (Medium) — ~14h conforme Sprint 5 do PRD.
**Story Points:** 3

## Scope

### IN

- Página `/dashboard/certificados` — listagem de certificados do aluno (mb.css)
- Página pública `/certificado/[uuid]` — visualização e verificação (mb.css)
- API route `GET /api/certificate/[uuid]` — dados do certificado para verificação
- API route `POST /api/certificate/generate` — geração manual (admin) ou automática
- Trigger automático: ao marcar `enrollment.status = 'completed'`, gerar certificado se nota ≥ 0 (sem quiz = 10.0)
- Supabase Storage: salvar PDF gerado em bucket `certificates`
- Registro em tabela `certificates` com UUID, `pdf_url`, `issued_at`
- QR Code apontando para `/certificado/[uuid]`
- Botão "Baixar PDF" + "Imprimir"
- Estado vazio quando aluno não tem certificados

### OUT

- Email automático de certificado emitido (Sprint 6 — Resend)
- Painel admin de certificados (Sprint 6)
- Geração via Supabase Edge Function (simplificado: gerar na API route)
- Restrição por nota mínima (sem quiz = 10.0, com quiz usa score da última tentativa aprovada)

## Dependencies

- Story 3.2 Done: quiz_attempts com score disponível
