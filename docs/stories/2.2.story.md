## Story

**As a** aluno autenticado na plataforma EAD Mentoria Fábio Borges,
**I want** um dashboard pessoal com meus cursos matriculados, progresso visual, dados do meu perfil e informações do meu plano ativo,
**so that** eu possa acompanhar meu aprendizado, manter meus dados atualizados e entender os benefícios do meu plano de mentoria.

## Business Value

O dashboard é a "sala de aula" do aluno — a primeira tela que ele vê após logar. Um dashboard bem estruturado aumenta retenção: o aluno vê progresso, sente evolução e permanece engajado. Sem ele, o aluno fica perdido após o login mesmo tendo pago pelo conteúdo.

**IN (o que esta story entrega):**

### Query de Progresso por Curso

Para calcular progresso sem N+1 queries, usar uma query com agregação:

```sql
-- Aulas concluídas por curso
SELECT
  e.course_id,
  COUNT(lp.id) FILTER (WHERE lp.completed = true) as completed_lessons,
  COUNT(l.id) as total_lessons
FROM enrollments e
JOIN modules m ON m.course_id = e.course_id
JOIN lessons l ON l.module_id = m.id
LEFT JOIN lesson_progress lp ON lp.lesson_id = l.id AND lp.user_id = e.user_id
WHERE e.user_id = auth.uid() AND e.status = 'active'
GROUP BY e.course_id
```

Como alternativa mais simples no cliente Supabase:

```typescript
// Para cada enrollment: buscar total de lições e total completas
const { count: total } = await supabase
  .from('lessons')
  .select('id', { count: 'exact' })
  .in('module_id', moduleIds)

const { count: completed } = await supabase
  .from('lesson_progress')
  .select('id', { count: 'exact' })
  .eq('user_id', userId)
  .in('lesson_id', lessonIds)
  .eq('completed', true)
```
