## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** poder definir o preço parcelado total e o número de parcelas ao criar ou editar um curso,
**so that** eu possa habilitar parcelamento para cursos específicos com valor calculado em tempo real.

## Business Value

O admin precisa de controle granular sobre precificação: não todos os cursos devem ter parcelamento (cursos baratos não precisam, cursos premium sim). O cálculo em tempo real do valor da parcela elimina erros manuais de divisão e dá ao Fábio visibilidade imediata do que o aluno vai ver na página do curso.

## Complexity Estimate

**T-shirt:** S (Small) — ~3h
**Story Points:** 2

## Scope

### IN

- No formulário de criação/edição de curso em `/admin/cursos/`:
  - Campo "Preço parcelado total (R$)" → `price_installment` (nullable — deixar em branco = sem parcelamento)
  - Campo "Número máximo de parcelas" → `max_installments` (número inteiro ≥ 1; default 1)
  - Display calculado em tempo real: "Valor da parcela: R$ X,XX" (= `price_installment / max_installments`)
  - Validação: `price_installment` deve ser ≥ `price` (preço à vista) quando preenchido
  - Validação: `max_installments` ≥ 1 (não pode ser zero ou negativo)

### OUT

- Exibição no frontend de curso (Story 20.3)
- Integração com Stripe para parcelamento (Story 20.4)
- Parcelamento para planos de assinatura (não aplicável — apenas para cursos avulsos)

## Dependencies

- **Story 20.1 deve estar completa** — colunas `price_installment` e `max_installments` devem existir
- Formulário de admin de curso existente (verificar caminho: `src/app/admin/cursos/`)
- Padrão de formulário existente (React Hook Form, Zod, ou state simples — verificar padrão atual)

## Acceptance Criteria

- **AC-1:** No formulário de edição de curso, os campos "Preço parcelado total" e "Número de parcelas" estão presentes
- **AC-2:** Ao preencher ambos os campos, o texto "Valor da parcela: R$ X,XX" é atualizado em tempo real (sem submit)
- **AC-3:** Ao salvar o curso com `price_installment = R$300,00` e `max_installments = 3`, o banco registra `price_installment = 30000` (centavos) e `max_installments = 3`
- **AC-4:** Se `price_installment` for preenchido com valor menor que `price` (preço à vista), o formulário exibe validação de erro: "Preço parcelado não pode ser menor que o preço à vista" e bloqueia o submit
- **AC-5:** Se `max_installments` for 0 ou negativo, o formulário exibe erro de validação e bloqueia o submit
- **AC-6:** Deixar `price_installment` em branco é válido — salva `NULL` no banco (curso sem parcelamento)
- **AC-7:** Ao reabrir o formulário de edição de um curso com parcelamento configurado, os campos exibem os valores atuais

### Lógica de cálculo em tempo real

```typescript
// Calcular valor da parcela em tempo real
const installmentValue = useMemo(() => {
  if (!priceInstallment || !maxInstallments || maxInstallments <= 0) return null
  return priceInstallment / maxInstallments
}, [priceInstallment, maxInstallments])

// Exibir
{installmentValue && maxInstallments > 1 && (
  <p>Valor da parcela: {formatCurrency(installmentValue)}</p>
)}
```

### Validação de coerência de preços

```typescript
// Validação: price_installment >= price (à vista)
const validatePriceInstallment = (value: number | null, coursePrice: number) => {
  if (value === null || value === undefined) return true // campo opcional
  if (value < coursePrice) return 'Preço parcelado não pode ser menor que o preço à vista'
  return true
}
```

### Formatação de valores

Os preços no banco são em centavos (integer). O formulário deve:

- Exibir em reais: `price_installment / 100` → formatar como "R$ 300,00"
- Salvar em centavos: `valor_em_reais * 100`

Verificar o padrão de input de moeda existente no formulário de preço à vista (`price`) — replicar para os novos campos.

### Payload para salvar

```typescript
// Incluir no payload de update do curso:
{
  price_installment: priceInstallmentInput ? Math.round(priceInstallmentInput * 100) : null,
  max_installments: maxInstallmentsInput || 1,
}
```
