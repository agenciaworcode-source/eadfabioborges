# Story 14.4: Testes — Validação Multi-tipo

## Story

**As a** desenvolvedor da plataforma,
**I want** testes automatizados que cubram cada branch de tipo de aula no player e no editor,
**so that** regressões em qualquer tipo de aula sejam detectadas antes de chegar à produção.

## Business Value

As stories 14.1–14.3 introduzem 4 caminhos de código distintos no player e no editor. Sem testes, qualquer alteração futura (ex: novo campo, refactor do switch) pode silenciosamente quebrar um tipo. Dado que os cursos publicados têm aulas de vídeo com alunos ativos, o risco de regressão é real e os testes são a rede de segurança.

## Complexity Estimate

**T-shirt:** S (Small) — ~3h
**Story Points:** 2

## Scope

### IN

#### Testes unitários — LessonContent / CoursePlayerLayout (Vitest + React Testing Library)

- Teste: tipo `video` → renderiza elemento de player de vídeo (não renderiza iframe/texto)
- Teste: tipo `text` com `content_body` preenchido → renderiza o HTML do conteúdo
- Teste: tipo `pdf` com `pdf_url` preenchido → renderiza `<iframe>` com o src correto
- Teste: tipo `embed` com `embed_url` preenchido → renderiza `<iframe>` com o src correto
- Teste: tipo `pdf` sem `pdf_url` → renderiza mensagem de fallback "Conteúdo indisponível"
- Teste: tipo `embed` sem `embed_url` → renderiza mensagem de fallback "Conteúdo indisponível"

#### Testes unitários — LessonEditor (Vitest + React Testing Library)

- Teste: selecionar tipo "Texto" → campo `content_body` aparece, `vimeo_id` desaparece
- Teste: selecionar tipo "PDF" → campo `pdf_url` aparece, `vimeo_id` e `content_body` desaparecem
- Teste: selecionar tipo "Embed" → campo `embed_url` aparece; ao preencher a URL, o iframe de preview é renderizado
- Teste: selecionar tipo "Vídeo" → campo `vimeo_id` aparece, demais campos de conteúdo desaparecem

#### Verificação de build e lint

- `npm test` deve passar sem falhas
- `npm run build`, `npm run lint`, `npm run typecheck` sem erros

### OUT

- Testes end-to-end com Playwright ou Cypress
- Testes de integração com banco de dados real
- Testes de acessibilidade automática
- Cobertura de 100% de branches — focar nos caminhos críticos
