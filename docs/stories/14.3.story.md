## Story

**As a** aluno da plataforma,
**I want** visualizar aulas de vídeo, texto, PDF e embed sem diferença de experiência,
**so that** o conteúdo do curso seja acessível independente do formato escolhido pelo instrutor.

## Business Value

Após as stories 14.1 e 14.2, os dados existem no banco e o admin consegue criar aulas multi-tipo. Porém o `CoursePlayerLayout` continua renderizando apenas `<VimeoPlayer>`. Um aluno que acessar uma aula de texto, PDF ou embed verá o player de vídeo tentando carregar sem `vimeo_id` — experiência quebrada. Esta story fecha o ciclo para o aluno.

## Complexity Estimate

**T-shirt:** M (Medium) — ~4h
**Story Points:** 3

## Scope

### IN

#### CoursePlayerLayout — renderização por tipo

- Substituir a renderização fixa de `<VimeoPlayer>` por um switch condicional baseado em `lesson.type`:
  - `'video'`: renderizar `<VimeoPlayer>` (comportamento atual, sem mudança)
  - `'text'`: renderizar `content_body` como HTML seguro usando `dangerouslySetInnerHTML`
  - `'pdf'`: renderizar `<iframe>` apontando para `pdf_url` com dimensões adequadas
  - `'embed'`: renderizar `<iframe>` apontando para `embed_url` com dimensões adequadas

#### Atualizar interfaces de tipos

- Expandir `LessonRow` local de `CoursePlayerLayout.tsx` para incluir `type`, `content_body`, `embed_url`, `pdf_url`

#### Progresso automático (marcar aula como concluída)

- Para tipo `'video'`: manter a lógica atual via callback `onComplete` do `VimeoPlayer`
- Para tipos `'text'`, `'pdf'`, `'embed'`: marcar aula como concluída ao visualizar (ao selecionar a aula), pois não há evento de "fim de reprodução"

#### Atualizar query de dados do curso

- O Server Component (`src/app/(dashboard)/dashboard/curso/[id]/page.tsx`) busca os campos de lessons. Adicionar `type`, `content_body`, `embed_url`, `pdf_url` no SELECT do Supabase para que os dados cheguem ao `CoursePlayerLayout`

### OUT

- Sanitização avançada de HTML (DOMPurify ou similar) — `dangerouslySetInnerHTML` é aceitável pois o conteúdo é inserido pelo admin autenticado, não por usuários finais
- Player de PDF com controles avançados (zoom, página, download block)
- Fullscreen de embed
- Ícone diferenciado por tipo no rail lateral

## Acceptance Criteria

- **AC-1:** Aula do tipo `video` continua renderizando `<VimeoPlayer>` sem nenhuma mudança de comportamento
- **AC-2:** Aula do tipo `text` renderiza o conteúdo de `content_body` como HTML dentro da área de conteúdo principal (onde o player ficaria)
- **AC-3:** Aula do tipo `pdf` renderiza um `<iframe>` com `src={lesson.pdf_url}` ocupando a área de conteúdo principal com altura mínima de 600px
- **AC-4:** Aula do tipo `embed` renderiza um `<iframe>` com `src={lesson.embed_url}` com proporção 16:9 (aspect-ratio)
- **AC-5:** Aula do tipo `text`, `pdf` ou `embed` é marcada como concluída automaticamente ao ser selecionada pelo aluno (chamada à API de progresso ocorre ao selecionar a aula)
- **AC-6:** O rail lateral (lista de aulas) exibe a marcação de concluída corretamente para aulas de todos os tipos
- **AC-7:** Quando `pdf_url` ou `embed_url` estiver vazia/nula, o player exibe mensagem de fallback: "Conteúdo indisponível"
- **AC-8:** `npm run build`, `npm run lint` e `npm run typecheck` passam sem erros

### Switch de renderização do conteúdo principal

```tsx
// Substituir o bloco que renderiza <VimeoPlayer> na seção {/* Player */}
function LessonContent({
  lesson,
  courseId,
  hasAccess,
  lessons,
  nextLesson,
  selectLesson,
}: {
  lesson: LessonRow
  courseId: string
  hasAccess: boolean
  lessons: Record<string, { completed: boolean; watchedSecs: number }>
  nextLesson: LessonRow | null
  selectLesson: (id: string) => void
}) {
  const lessonType = lesson.type ?? 'video'

  if (!hasAccess) {
    return <LockedContent /> // componente já existente
  }

  switch (lessonType) {
    case 'video':
      return (
        <VimeoPlayer
          key={lesson.id}
          vimeoId={lesson.vimeo_id}
          lessonId={lesson.id}
          courseId={courseId}
          durationSecs={lesson.duration_secs}
          initialWatchedSecs={lessons[lesson.id]?.watchedSecs ?? 0}
          onComplete={() => {
            if (nextLesson) selectLesson(nextLesson.id)
          }}
        />
      )
    case 'text':
      return (
        <div
          className="prose max-w-none p-6"
          style={{ background: '#fff', minHeight: '400px' }}
          dangerouslySetInnerHTML={{ __html: lesson.content_body ?? '' }}
        />
      )
    case 'pdf':
      if (!lesson.pdf_url) return <FallbackContent message="Conteúdo indisponível" />
      return (
        <iframe
          src={lesson.pdf_url}
          style={{ width: '100%', height: '600px', border: 'none' }}
          title={lesson.title}
        />
      )
    case 'embed':
      if (!lesson.embed_url) return <FallbackContent message="Conteúdo indisponível" />
      return (
        <div style={{ position: 'relative', paddingBottom: '56.25%', height: 0 }}>
          <iframe
            src={lesson.embed_url}
            style={{
              position: 'absolute',
              top: 0,
              left: 0,
              width: '100%',
              height: '100%',
              border: 'none',
            }}
            title={lesson.title}
            allowFullScreen
          />
        </div>
      )
    default:
      return <FallbackContent message="Tipo de aula desconhecido" />
  }
}
```

### Marcar aula como concluída para tipos não-vídeo

Ao selecionar uma aula (via `selectLesson`), se o tipo não for `'video'`, chamar a API de progresso para marcar como concluída:

```typescript
// Localizar ou verificar: src/hooks/use-progress.ts ou src/app/api/progress/route.ts
// A chamada existente é POST /api/progress com { lessonId, courseId, completed: true }
// Disparar ao selecionar aula não-vídeo:

async function selectLesson(id: string) {
  setSelectedLessonId(id)
  setQuizResult(null)
  const lesson = allLessons.find((l) => l.id === id)
  if (lesson && lesson.type !== 'video' && !lessons[id]?.completed) {
    // chamar API de progresso para marcar como vista
    await fetch('/api/progress', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ lessonId: id, courseId, completed: true, watchedSecs: 0 }),
    })
  }
}
```

**Nota:** Verificar a assinatura exata da API `/api/progress` antes de implementar. A lógica pode estar em `useCourseProgress` ou diretamente via fetch.

### Fallback para aulas sem vimeo_id no tipo vídeo

Aulas antigas com `type = 'video'` mas `vimeo_id = null` devem exibir mensagem de fallback ao invés de travar o player:

```tsx
case 'video':
  if (!lesson.vimeo_id) return <FallbackContent message="Vídeo não configurado" />
  return <VimeoPlayer ... />
```
