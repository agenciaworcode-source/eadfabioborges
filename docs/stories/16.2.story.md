## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** gerenciar quais cursos estão inclusos em cada plano diretamente pela interface administrativa,
**so that** eu possa adicionar ou remover cursos de um plano sem precisar editar SQL diretamente no banco.

## Business Value

Sem essa UI, a tabela `plan_courses` criada na 16.1 só pode ser gerenciada via SQL — tornando o Epic 16 inutilizável na prática. O admin precisa de autonomia para montar os pacotes de cada plano conforme a estratégia comercial mude, sem depender de um desenvolvedor para cada alteração.

## Scope

### IN

#### Página admin de planos — aba "Cursos inclusos"

- Na página existente `src/app/(admin)/admin/planos/page.tsx`: adicionar seção ou aba "Cursos inclusos" para cada plano
- Exibir checklist de todos os cursos publicados, marcados conforme `plan_courses` atual
- Admin pode marcar/desmarcar cursos para incluir/remover de um plano
- Botão "Salvar alterações" que persiste as mudanças via API

#### API de gestão de cursos por plano

- `POST /api/admin/plans/[planId]/courses` — adiciona um curso ao plano
  - Body: `{ courseId: string }`
  - Valida: plano existe e está ativo; curso existe e está publicado
  - Retorna: 201 Created ou 409 Conflict se já existe
- `DELETE /api/admin/plans/[planId]/courses/[courseId]` — remove um curso do plano
  - Retorna: 204 No Content ou 404 se não existe

#### Feedback visual

- Loading state durante chamadas de API
- Toast de sucesso após salvar
- Toast de erro se a API retornar falha
- Exibir contagem: "X cursos inclusos neste plano"

### OUT

- Criação ou edição de planos em si (já existe na página de planos)
- Drag-and-drop para ordenar cursos dentro do plano
- Histórico de alterações (quem adicionou/removeu qual curso)
- Endpoint bulk (adicionar múltiplos cursos em uma única chamada — usa o botão "Salvar" que faz chamadas individuais)
- Testes automatizados de componentes

## Technical Notes

### Estrutura de rotas de API

```
src/app/api/admin/plans/[planId]/courses/route.ts       → POST (adicionar)
src/app/api/admin/plans/[planId]/courses/[courseId]/route.ts → DELETE (remover)
```

### Verificação de admin nas rotas

```typescript
// Padrão a seguir em cada route handler
const supabase = createClient()
const {
  data: { user },
} = await supabase.auth.getUser()
if (!user) return NextResponse.json({ error: 'Não autenticado' }, { status: 401 })

const { data: profile } = await supabase.from('users').select('role').eq('id', user.id).single()
if (profile?.role !== 'admin') return NextResponse.json({ error: 'Acesso negado' }, { status: 403 })
```

### Query para carregar estado inicial

```typescript
// Buscar todos os cursos publicados + quais estão no plano
const { data: courses } = await supabase
  .from('courses')
  .select('id, title')
  .eq('published', true)
  .order('title')

const { data: planCourses } = await supabase
  .from('plan_courses')
  .select('course_id')
  .eq('plan_id', planId)

const includedIds = new Set(planCourses?.map((pc) => pc.course_id) ?? [])
```
