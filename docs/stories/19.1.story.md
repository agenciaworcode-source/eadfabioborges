## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** que o banco suporte a opção de billing semestral com seu próprio preço por plano,
**so that** as stories 19.2 e 19.3 tenham onde persistir e consultar os preços semestrais.

## Business Value

O billing semestral é um ponto de conversão intermediário: alunos que hesitam no anual (comprometimento de 12 meses) podem preferir 6 meses com desconto sobre o mensal. Esse pacing de preço reduz a barreira de entrada e aumenta o LTV médio. Sem a foundation no banco (coluna + enum), nenhuma das outras stories do Epic 19 pode ser implementada.

## Scope

### IN

#### Coluna `price_semiannual`

- `ALTER TABLE public.plans ADD COLUMN IF NOT EXISTS price_semiannual integer NOT NULL DEFAULT 0`
- Valor em centavos, consistente com as colunas `price_monthly` e `price_annual` existentes
- Seed: preencher `price_semiannual` para os planos existentes com ~70% do preço anual (arredondado para cima ao inteiro mais próximo)

#### Enum `subscription_plan`

- `ALTER TYPE public.subscription_plan ADD VALUE IF NOT EXISTS 'semiannual'`
- Verificar se o tipo `subscription_plan` é ENUM no banco (verificar migration do Epic 16)

#### Coluna `stripe_price_id_semiannual` na tabela `plans`

- `ALTER TABLE public.plans ADD COLUMN IF NOT EXISTS stripe_price_id_semiannual text`
- Nullable (Stripe Price ID será configurado pelo admin quando necessário)

### OUT

- Criação de Stripe Prices semestrais (responsabilidade do admin/Story 19.3)
- UI para exibir opção semestral (Story 19.2)
- Checkout semestral (Story 19.3)
- Alteração no modelo de dados de `subscriptions` (o campo `billing_period` já suporta valores de texto)

## Acceptance Criteria

- **AC-1:** Após aplicar a migration, `\d plans` mostra a coluna `price_semiannual integer NOT NULL DEFAULT 0`

- **AC-7:** Executar a migration duas vezes (idempotência via `IF NOT EXISTS` e `ADD VALUE IF NOT EXISTS`) não gera erros

## Technical Notes

### Verificação prévia obrigatória

Antes de gerar a migration, o dev deve verificar:

```sql
-- Verificar tipo de subscription_plan
SELECT typname, typtype FROM pg_type WHERE typname = 'subscription_plan';
-- typtype = 'e' → é ENUM (usar ADD VALUE IF NOT EXISTS)
-- typtype = 'b' ou ausente → é texto simples (ADD VALUE não se aplica)

-- Verificar nomes de colunas de preço em plans
SELECT column_name FROM information_schema.columns
WHERE table_name = 'plans'
ORDER BY ordinal_position;
```

### SQL da migration (template)

```sql
-- Migration: Epic 19.1 — Suporte Semestral

-- Adicionar coluna de preço semestral
ALTER TABLE public.plans
  ADD COLUMN IF NOT EXISTS price_semiannual integer NOT NULL DEFAULT 0;

-- Adicionar coluna de Stripe Price ID semestral
ALTER TABLE public.plans
  ADD COLUMN IF NOT EXISTS stripe_price_id_semiannual text;

-- Adicionar valor 'semiannual' ao enum (se subscription_plan for ENUM)
-- Verificar tipo antes de incluir esta linha
DO $$
BEGIN
  IF EXISTS (SELECT 1 FROM pg_type WHERE typname = 'subscription_plan' AND typtype = 'e') THEN
    ALTER TYPE public.subscription_plan ADD VALUE IF NOT EXISTS 'semiannual';
  END IF;
END$$;

-- Seed: preencher price_semiannual (~70% do anual, arredondado para cima)
-- EXEMPLO (ajustar valores reais conforme plans existentes):
UPDATE public.plans SET price_semiannual = CEIL(price_annual * 0.70)
WHERE price_annual > 0 AND price_semiannual = 0;
```

### Nomenclatura do arquivo

`supabase/migrations/YYYYMMDDHHMMSS_epic19_semiannual.sql`

### Seed de preços (referência)

Se os planos atuais tiverem preços anuais como:

- prata: R$199/ano → semestral sugerido: ~R$140 (CEIL(199 \* 0.70) = 140)
- ouro: R$299/ano → semestral sugerido: ~R$210
- diamante: R$499/ano → semestral sugerido: ~R$350

Os valores exatos dependem dos dados reais da tabela `plans` — o dev deve confirmar antes do seed.
