## Story

**As a** plataforma (sistema),
**I want** processar automaticamente os eventos do Stripe e do MercadoPago (PIX) via webhooks,
**so that** quando um pagamento for confirmado, a matrícula ou assinatura seja criada no banco de dados sem intervenção manual, liberando o acesso do aluno imediatamente.

## Business Value

O checkout Stripe (Story 11.1) cria a sessão e redireciona o aluno. Mas sem o webhook, nenhuma matrícula é criada no banco — o aluno paga e fica sem acesso. O webhook é a ponte crítica entre "pagamento aprovado" e "acesso liberado". Esta story também adiciona o provider PIX (MercadoPago) para pagamentos únicos em BRL — fundamental para o público brasileiro que prefere PIX.

## Acceptance Criteria

- **AC-1:** POST `/api/webhooks/stripe` com assinatura HMAC inválida retorna 400 ("Invalid signature")
- **AC-2:** POST `/api/webhooks/stripe` com evento `checkout.session.completed` (type=course) válido cria linha em `enrollments` com `status='active'`; se já existir, não duplica (idempotência)
- **AC-3:** POST `/api/webhooks/stripe` com evento `checkout.session.completed` (type=plan) válido cria linha em `subscriptions` com `status='active'`, `plan=planId`, `period_start/period_end` da subscription Stripe
- **AC-4:** POST `/api/webhooks/stripe` com evento `customer.subscription.deleted` atualiza `subscriptions.status = 'canceled'` para o `stripe_sub_id` correspondente
- **AC-5:** POST `/api/webhooks/mercadopago` com `x-signature` inválida retorna 400
- **AC-6:** POST `/api/webhooks/mercadopago` com notificação de pagamento `status=approved` cria enrollment ou subscription conforme `external_reference` (formato: `course:{courseId}:{userId}` ou `plan:{planId}:{userId}:{billingPeriod}`)
- **AC-7:** Todos os INSERTs usam `createServiceClient()` — nunca o client de sessão (webhooks não têm sessão de usuário)
- **AC-8:** Erros de banco nos handlers retornam 500 com `{ error }` mas não travam outros eventos do mesmo batch Stripe (cada handler tem try/catch independente)
- **AC-9:** `npm run lint` e `npm run typecheck` sem erros

## Technical Notes

### Webhook Stripe — verificação de assinatura

```typescript
// src/app/api/webhooks/stripe/route.ts
import { NextResponse } from 'next/server'
import Stripe from 'stripe'
import { createServiceClient } from '@/lib/supabase/service'

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!, { apiVersion: '2024-06-20' })

export async function POST(request: Request) {
  const body = await request.text() // CRÍTICO: text(), não json()
  const sig = request.headers.get('stripe-signature')

  let event: Stripe.Event
  try {
    event = stripe.webhooks.constructEvent(body, sig!, process.env.STRIPE_WEBHOOK_SECRET!)
  } catch {
    return NextResponse.json({ error: 'Invalid signature' }, { status: 400 })
  }

  const supabase = createServiceClient()

  if (event.type === 'checkout.session.completed') {
    const session = event.data.object as Stripe.Checkout.Session
    const { type, userId, courseId, planId, billingPeriod } = session.metadata ?? {}

    if (type === 'course' && userId && courseId) {
      // Idempotência
      const { data: existing } = await supabase
        .from('enrollments')
        .select('id')
        .eq('user_id', userId)
        .eq('course_id', courseId)
        .maybeSingle()

      if (!existing) {
        await supabase.from('enrollments').insert({
          id: crypto.randomUUID(),
          user_id: userId,
          course_id: courseId,
          status: 'active',
          enrolled_at: new Date().toISOString(),
        } as never)
      }
    }

    if (type === 'plan' && userId && planId) {
      const subscription =
        typeof session.subscription === 'string'
          ? await stripe.subscriptions.retrieve(session.subscription)
          : (session.subscription as Stripe.Subscription)

      const { data: existing } = await supabase
        .from('subscriptions')
        .select('id')
        .eq('user_id', userId)
        .eq('stripe_sub_id', subscription.id)
        .maybeSingle()

      if (!existing) {
        await supabase.from('subscriptions').insert({
          id: crypto.randomUUID(),
          user_id: userId,
          plan: planId,
          status: 'active',
          stripe_sub_id: subscription.id,
          period_start: new Date(subscription.current_period_start * 1000).toISOString(),
          period_end: new Date(subscription.current_period_end * 1000).toISOString(),
        } as never)

        // Atualizar users.plan para refletir assinatura ativa
        await supabase
          .from('users')
          .update({ plan: planId } as never)
          .eq('id', userId)
      }
    }
  }

  if (event.type === 'customer.subscription.updated') {
    const sub = event.data.object as Stripe.Subscription
    await supabase
      .from('subscriptions')
      .update({
        status: sub.status,
        period_end: new Date(sub.current_period_end * 1000).toISOString(),
      } as never)
      .eq('stripe_sub_id', sub.id)
  }

  if (event.type === 'customer.subscription.deleted') {
    const sub = event.data.object as Stripe.Subscription
    await supabase
      .from('subscriptions')
      .update({ status: 'canceled' } as never)
      .eq('stripe_sub_id', sub.id)
  }

  return NextResponse.json({ received: true })
}
```

**CRÍTICO:** `await request.text()` — Stripe rejeita se o body for parseado como JSON antes da verificação de assinatura.

### MercadoPago provider

```typescript
// src/lib/checkout/providers/mercadopago.ts
import { MercadoPagoConfig, Preference } from 'mercadopago'
import type { CheckoutProvider } from '../types'

const mp = new MercadoPagoConfig({ accessToken: process.env.MERCADOPAGO_ACCESS_TOKEN! })

export const mercadoPagoProvider: CheckoutProvider = {
  async createCourseCheckout(params) {
    const preference = new Preference(mp)
    const result = await preference.create({
      body: {
        items: [
          {
            title: params.courseTitle,
            quantity: 1,
            unit_price: params.priceAmountCents / 100,
            currency_id: 'BRL',
          },
        ],
        external_reference: `course:${params.courseId}:${params.userId}`,
        redirect_urls: {
          success: params.successUrl,
          failure: params.cancelUrl,
          pending: params.successUrl,
        },
        payment_methods: { excluded_payment_types: [] }, // permite PIX
      },
    })
    return { url: result.init_point! }
  },
  async createPlanCheckout(params) {
    // MercadoPago não tem subscription nativa — gerar pagamento único pelo preço do período
    const price =
      params.billingPeriod === 'annual' ? params.priceAmountCents * 12 : params.priceAmountCents
    const preference = new Preference(mp)
    const result = await preference.create({
      body: {
        items: [
          {
            title: `${params.planName} (${params.billingPeriod === 'annual' ? 'Anual' : 'Mensal'})`,
            quantity: 1,
            unit_price: price / 100,
            currency_id: 'BRL',
          },
        ],
        external_reference: `plan:${params.planId}:${params.userId}:${params.billingPeriod}`,
        redirect_urls: {
          success: params.successUrl,
          failure: params.cancelUrl,
          pending: params.successUrl,
        },
      },
    })
    return { url: result.init_point! }
  },
}
```
