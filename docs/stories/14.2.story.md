## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** selecionar o tipo de uma aula (vídeo, texto, PDF ou embed) no editor do admin e preencher os campos correspondentes,
**so that** eu possa criar aulas com formatos além do Vimeo e salvar o conteúdo correto para cada tipo.

## Business Value

O `LessonEditor` atual só tem o campo `vimeo_id`. Com o schema expandido da 14.1, o admin precisa de um seletor de tipo e campos condicionais para cada formato — de outra forma o Fábio não consegue criar aulas que não sejam de vídeo, mesmo após a migration estar aplicada.

## Complexity Estimate

**T-shirt:** M (Medium) — ~4h
**Story Points:** 3

## Scope

### IN

#### LessonEditor — Seletor de tipo

- Adicionar campo `type` (radio buttons ou select) com as 4 opções: `Vídeo`, `Texto`, `PDF`, `Embed`
- O campo `type` tem valor padrão `video` (alinhado com o DEFAULT do banco)

#### Campos condicionais por tipo

- **Vídeo** (`type === 'video'`): exibir campo `vimeo_id` (já existente — sem alteração)
- **Texto** (`type === 'text'`): exibir `<textarea>` para `content_body` (HTML ou texto livre)
- **PDF** (`type === 'pdf'`): exibir campo de URL para `pdf_url` (input text, placeholder com exemplo de URL)
- **Embed** (`type === 'embed'`): exibir campo de URL para `embed_url` + preview inline do embed via `<iframe>` quando URL preenchida

#### Atualizar chamadas de API existentes

- POST `/api/admin/modules/[id]/lessons` — enviar `type`, `content_body`, `embed_url`, `pdf_url` junto com os campos já existentes
- PATCH `/api/admin/lessons/[id]` — idem
- Atualizar os Zod schemas dos dois endpoints para aceitar os novos campos

#### Atualizar tipo `LessonRow`

- Adicionar `type`, `content_body`, `embed_url`, `pdf_url` na interface `LessonRow` exportada de `LessonEditor.tsx`

#### Exibição na lista de aulas do módulo

- Mostrar badge de tipo na linha da aula (ex: `VIDEO`, `TEXTO`, `PDF`, `EMBED`) ao lado do título

### OUT

- Upload direto de arquivos PDF para Supabase Storage (campo aceita URL — o upload é responsabilidade externa)
- Editor rich-text (TipTap, Quill ou similar) — apenas textarea nesta story
- Preview de conteúdo texto no admin
- Qualquer mudança no player (fica para Story 14.3)

## Acceptance Criteria

- **AC-1:** O formulário de adicionar/editar aula exibe um seletor de tipo com as opções: Vídeo, Texto, PDF, Embed
- **AC-2:** Ao selecionar "Vídeo", apenas o campo `vimeo_id` é exibido (comportamento atual preservado)
- **AC-3:** Ao selecionar "Texto", apenas o campo `content_body` (textarea) é exibido; `vimeo_id` e os campos de URL ficam ocultos
- **AC-4:** Ao selecionar "PDF", apenas o campo `pdf_url` (input text) é exibido
- **AC-5:** Ao selecionar "Embed", apenas o campo `embed_url` (input text) é exibido; quando a URL está preenchida, um preview via `<iframe>` aparece abaixo do campo
- **AC-6:** Salvar uma aula do tipo "Texto" persiste `type = 'text'` e `content_body` no banco; `vimeo_id` e demais campos de URL ficam `null`
- **AC-7:** Salvar uma aula do tipo "PDF" persiste `type = 'pdf'` e `pdf_url` no banco
- **AC-8:** Salvar uma aula do tipo "Embed" persiste `type = 'embed'` e `embed_url` no banco
- **AC-9:** Editar uma aula existente carrega o `type` correto e exibe os campos do tipo correspondente pré-preenchidos
- **AC-10:** A linha da aula na lista do módulo exibe um badge indicando o tipo (`VIDEO`, `TEXTO`, `PDF`, `EMBED`)
- **AC-11:** `npm run build`, `npm run lint` e `npm run typecheck` passam sem erros

### Atualização dos Zod schemas das APIs

```typescript
// src/app/api/admin/lessons/[id]/route.ts — patchSchema
const patchSchema = z
  .object({
    title: z.string().min(1).optional(),
    type: z.enum(['video', 'text', 'pdf', 'embed']).optional(),
    vimeo_id: z.string().nullable().optional(),
    content_body: z.string().nullable().optional(),
    embed_url: z.string().nullable().optional(),
    pdf_url: z.string().nullable().optional(),
    duration_secs: z.number().int().min(0).optional(),
    order: z.number().int().min(0).optional(),
    is_free_preview: z.boolean().optional(),
  })
  .strict()

// src/app/api/admin/modules/[id]/lessons/route.ts — bodySchema
const bodySchema = z.object({
  title: z.string().min(1).max(255),
  type: z.enum(['video', 'text', 'pdf', 'embed']).optional().default('video'),
  vimeo_id: z.string().optional().nullable(),
  content_body: z.string().optional().nullable(),
  embed_url: z.string().optional().nullable(),
  pdf_url: z.string().optional().nullable(),
  duration_secs: z.number().int().nonnegative().optional(),
  order: z.number().int().nonnegative().optional(),
  is_free_preview: z.boolean().optional(),
})
```

### Lógica de limpeza de campos no handleSave

Ao salvar, enviar `null` nos campos que não pertencem ao tipo selecionado:

```typescript
// Exemplo: tipo 'text' → vimeo_id, embed_url, pdf_url = null
const payload = {
  title: form.title.trim(),
  type: form.type,
  vimeo_id: form.type === 'video' ? form.vimeo_id.trim() || null : null,
  content_body: form.type === 'text' ? form.content_body.trim() || null : null,
  embed_url: form.type === 'embed' ? form.embed_url.trim() || null : null,
  pdf_url: form.type === 'pdf' ? form.pdf_url.trim() || null : null,
  duration_secs: parseInt(form.duration_secs) || 0,
  is_free_preview: form.is_free_preview,
}
```
