## Story

**As a** visitante ou aluno da plataforma,
**I want** ver a opção de parcelamento na página do curso quando disponível,
**so that** eu possa tomar a decisão de compra sabendo o valor à vista e o parcelado antes de ir para o checkout.

## Business Value

Exibir o parcelamento na landing page do curso é uma das ações de conversão mais efetivas no e-commerce brasileiro: "12x de R$ 29,90" converte melhor que "R$ 299,00" para cursos de ticket médio/alto. A exibição condicional garante que cursos sem parcelamento configurado não mostrem informação incorreta.

## Complexity Estimate

**T-shirt:** XS (Extra Small) — ~1-2h
**Story Points:** 1

## Scope

### IN

- Em `/cursos/[slug]` (landing page do curso): lógica condicional de exibição de preço:
  - Se `max_installments > 1` e `price_installment IS NOT NULL`: exibir ambas as opções — "R$ X,XX à vista ou Xх de R$ Y,YY"
  - Se `max_installments = 1` ou `price_installment IS NULL`: exibir apenas preço à vista (comportamento atual sem alteração)
- Cálculo do valor da parcela: `price_installment / max_installments` (arredondado para 2 casas decimais)
- Formatação monetária em R$ (formato brasileiro)

### OUT

- Checkout com parcelamento (Story 20.4)
- Exibição de parcelamento em listas de cursos ou cards (apenas na landing page individual por ora)
- Badge "Parcelamento disponível" em cards de listagem

## Dependencies

- **Story 20.1 deve estar completa** — campos `price_installment` e `max_installments` devem existir no banco
- **Story 20.2 deve estar completa** (ou pelo menos um curso com dados de parcelamento para testar)
- Página `/cursos/[slug]` existente (verificar localização: `src/app/(public)/cursos/[slug]/page.tsx` ou similar)

## Acceptance Criteria

- **AC-1:** Para um curso com `max_installments = 3` e `price_installment = 30000` (R$300,00): a página exibe "R$ 150,00 à vista ou 3x de R$ 100,00"
- **AC-2:** Para um curso com `max_installments = 1` (default): a página exibe apenas o preço à vista — sem menção a parcelas
- **AC-3:** Para um curso com `price_installment IS NULL`: a página exibe apenas o preço à vista — sem menção a parcelas
- **AC-4:** O valor da parcela é calculado corretamente: `price_installment / max_installments` arredondado para 2 casas decimais
- **AC-5:** A formatação monetária usa o padrão R$ com vírgula decimal (ex: "R$ 100,00" e não "R$ 100.00")

## Technical Notes

### Arquivo a modificar

`src/app/(public)/cursos/[slug]/page.tsx` (ou localização equivalente — verificar estrutura de rotas públicas)

### Query ao banco

Certificar que a query que busca o curso inclui os novos campos:

```typescript
const { data: course } = await supabase
  .from('courses')
  .select('*, price, price_installment, max_installments') // garantir inclusão
  .eq('slug', slug)
  .single()
```

### Componente de exibição de preço

```typescript
// Lógica de exibição condicional
const hasInstallments = course.max_installments > 1 && course.price_installment != null

const installmentValue = hasInstallments
  ? course.price_installment / course.max_installments
  : null

// JSX
{hasInstallments ? (
  <p>
    {formatCurrency(course.price / 100)} à vista ou{' '}
    {course.max_installments}x de {formatCurrency(installmentValue! / 100)}
  </p>
) : (
  <p>{formatCurrency(course.price / 100)}</p>
)}
```

### Função de formatação monetária

Verificar se já existe utilitário de formatação no projeto (ex: `src/lib/utils.ts`, `src/utils/currency.ts`). Se existir, reutilizar. Se não existir, usar:

```typescript
const formatCurrency = (value: number) =>
  new Intl.NumberFormat('pt-BR', { style: 'currency', currency: 'BRL' }).format(value)
```

### Atenção: valores em centavos

- `course.price` está em centavos → dividir por 100 antes de formatar
- `course.price_installment` está em centavos → dividir por 100 antes de formatar
- `installmentValue` = `price_installment / max_installments` (ainda em centavos) → dividir por 100 para exibir
