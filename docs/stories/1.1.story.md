## Story

**As a** desenvolvedor do projeto EAD Mentoria Fábio Borges,
**I want** um repositório Next.js 14 completamente scaffoldado com TypeScript, Tailwind, ESLint, Prettier, Husky e estrutura de pastas padrão,
**so that** toda a equipe (e os agentes de desenvolvimento) possa trabalhar em um ambiente consistente, com qualidade de código garantida desde o primeiro commit.

## Business Value

Esta story é a **fundação de todas as outras 6 sprints**. Sem um scaffold limpo e um ambiente de CI funcional, nenhuma feature pode ser desenvolvida com segurança. O valor de negócio é direto: reduz retrabalho, elimina inconsistências de ambiente entre desenvolvedor/agente, e garante que o primeiro deploy em produção (VPS) seja possível ainda na Sprint 1.

### Debug Log References

- **shadcn v4 + Tailwind 3 incompatibility**: `npx shadcn@latest` instala v4 que usa oklch e `@import "tw-animate-css"` — incompatível com Tailwind 3.4.x. Resolvido: CSS convertido para HSL, imports removidos, variáveis adicionadas ao `tailwind.config.ts`.
- **Geist font error**: shadcn init sobrescreveu `layout.tsx` com Geist (não exportado em next@14.2.35). Resolvido: layout reescrito com `Poppins` (fonte do projeto per front-end-spec).
- **create-next-app naming**: `EAD-FABIOBORGES` rejeitado por npm naming. Resolvido: scaffold em subpasta `ead-fabioborges/` e arquivos copiados para raiz.
- **Husky git not found**: `npx husky init` falhou sem git repo. Resolvido: `git init` antes do husky init.
