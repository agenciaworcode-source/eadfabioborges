## Story

**As a** Fábio Borges (admin) e como visitante/aluno na página de planos,
**I want** que o admin possa editar o preço semestral de cada plano, e que a página pública `/planos` exiba a opção semestral no toggle de período,
**so that** visitantes possam ver e comparar as 3 opções de billing (mensal, semestral, anual) antes de assinar.

## Business Value

O valor do billing semestral só se manifesta quando o visitante pode vê-lo lado a lado com as outras opções. O toggle de 3 posições (mensal/semestral/anual) é o padrão de mercado em SaaS para apresentar opções de billing — reduz fricção de decisão e pode aumentar conversão para o semestral (ponto médio de comprometimento).

## Complexity Estimate

**T-shirt:** S (Small) — ~3h
**Story Points:** 2

## Scope

### IN

**Admin (`/admin/planos`):**

- Adicionar campo "Preço semestral (R$)" no formulário de edição de cada plano
- Salvar `price_semiannual` via UPDATE na tabela `plans`
- Exibir valor atual do campo ao abrir o formulário de edição

**Frontend público (`/planos`):**

- Adicionar terceira opção "Semestral" ao toggle de período (atualmente apenas "Mensal" / "Anual")
- Ao selecionar "Semestral": exibir `price_semiannual` nos cards de plano (formatado como R$/semestre)
- Manter comportamento atual para Mensal e Anual

### OUT

- Checkout semestral (Story 19.3)
- Desconto badge/label "X% off" no semestral (pode ser adicionado futuramente)
- Criação de Stripe Prices semestrais pelo admin via UI (Story 19.3)

## Dependencies

- **Story 19.1 deve estar completa** — coluna `price_semiannual` deve existir no banco
- Formulário de admin de planos existente (verificar caminho em `src/app/admin/planos/`)
- Toggle de período existente na página `/planos` (verificar componente atual)

## Acceptance Criteria

- **AC-1:** Em `/admin/planos`, ao editar um plano, o campo "Preço semestral" está presente e exibe o valor atual de `price_semiannual`
- **AC-2:** Admin salva novo valor de `price_semiannual` → `SELECT price_semiannual FROM plans WHERE id = ?` retorna o valor atualizado
- **AC-3:** Em `/planos`, o toggle de período exibe 3 opções: "Mensal", "Semestral", "Anual"
- **AC-4:** Ao selecionar "Semestral" no toggle, os cards dos planos exibem o `price_semiannual` correspondente formatado como "R$ X,XX / semestre"
- **AC-5:** As seleções "Mensal" e "Anual" continuam funcionando corretamente após a adição da opção semestral (sem regressão)
- **AC-6:** Se `price_semiannual = 0` para um plano, o card exibe "R$ 0,00 / semestre" (ou pode ocultar o plano da listagem semestral — decisão do dev)

### Toggle de 3 opções

```typescript
type BillingPeriod = 'monthly' | 'semiannual' | 'annual'

const [billingPeriod, setBillingPeriod] = useState<BillingPeriod>('monthly')

// Exibição do preço conforme período
const getPrice = (plan: Plan, period: BillingPeriod) => {
  if (period === 'monthly') return plan.price_monthly
  if (period === 'semiannual') return plan.price_semiannual
  return plan.price_annual
}

const getPriceLabel = (period: BillingPeriod) => {
  if (period === 'monthly') return '/ mês'
  if (period === 'semiannual') return '/ semestre'
  return '/ ano'
}
```

### Campo admin

No formulário existente, adicionar:

```typescript
// Campo de input numérico para price_semiannual
<input
  type="number"
  min="0"
  placeholder="Preço semestral em centavos"
  value={form.price_semiannual}
  onChange={(e) => setForm({ ...form, price_semiannual: Number(e.target.value) })}
/>
```

Ou se o formulário usa moeda formatada (R$), converter de centavos para reais na exibição e voltar ao salvar.
