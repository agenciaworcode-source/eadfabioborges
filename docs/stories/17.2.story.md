## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** criar, visualizar, editar e desativar cupons de desconto pela interface administrativa,
**so that** eu possa gerenciar promoções sem precisar acessar o banco de dados diretamente.

## Business Value

Sem uma interface de gerenciamento, cupons não têm utilidade prática — qualquer alteração exigiria intervenção técnica. O admin precisa criar cupons em tempo real durante campanhas (lançamento, black friday, parceiros), monitorar quantos usos cada cupom teve e desativar cupons vencidos rapidamente. Essa story entrega autonomia operacional completa sobre o sistema de cupons.

## Complexity Estimate

**T-shirt:** M (Medium) — ~5h
**Story Points:** 3

## Scope

### IN

#### Nova página `/admin/cupons`

- Listagem de todos os cupons (incluindo inativos — admin vê tudo)
- Tabela com colunas: Código, Tipo, Desconto, Uso (current_uses / max_uses), Válido até, Escopo, Status (ativo/inativo)
- Botão "Novo cupom" abre modal ou formulário de criação
- Botão de edição por linha abre modal ou formulário de edição
- Toggle de ativação/desativação por linha (altera `is_active`)

#### Formulário de cupom (criação e edição)

- Código: campo text, obrigatório, convertido automaticamente para UPPERCASE ao digitar
- Tipo de desconto: select com opções "Porcentagem (%)" e "Valor fixo (R$)"
- Valor do desconto: campo numérico, obrigatório, > 0; se tipo = porcentagem, máximo 100
- Máximo de usos: campo numérico opcional (vazio = ilimitado)
- Data de início: date picker, padrão = hoje
- Data de término: date picker opcional (vazio = sem expiração)
- Escopo: select com opções "Todos os produtos", "Curso específico", "Plano específico"
- Produto alvo: aparece quando escopo = "Curso específico" ou "Plano específico" — select dos cursos/planos existentes
- Botão "Salvar" / "Cancelar"

#### Menu de navegação admin

- Adicionar link "Cupons" no menu lateral ou de navegação da área admin

### OUT

- Histórico de quem usou cada cupom (relatórios de uso por usuário)
- Importação em lote de cupons via CSV
- Geração automática de código aleatório (admin digita o código)
- Exclusão de cupons (apenas desativação via toggle `is_active`)
- Testes automatizados de componentes

## Acceptance Criteria

- **AC-1:** A página `/admin/cupons` existe e é acessível para usuário admin
- **AC-2:** A listagem exibe todos os cupons (ativos e inativos), incluindo colunas: Código, Desconto (ex: "10%" ou "R$ 50,00"), Usos (ex: "3 / 100" ou "3 / ∞"), Válido até (ou "Sem expiração"), Status
- **AC-3:** Admin preenche o formulário de criação e submete — o cupom aparece no banco com todos os campos corretos (`SELECT * FROM coupons WHERE code = 'NOVO10'`)
- **AC-4:** O código do cupom é automaticamente convertido para UPPERCASE: digitar "bemvindo" resulta em "BEMVINDO" ao salvar
- **AC-5:** Admin edita um cupom existente (ex: altera o `discount_value`) — a mudança persiste no banco
- **AC-6:** Admin clica no toggle de status de um cupom ativo — `is_active` muda para `false` no banco e a listagem reflete o novo status imediatamente
- **AC-7:** Tentar criar um cupom com código já existente exibe mensagem de erro "Código de cupom já existe"
- **AC-8:** Campo de valor do desconto com tipo "porcentagem" e valor > 100 exibe validação "Porcentagem não pode ser superior a 100%"
- **AC-9:** Menu de navegação da área admin inclui link para `/admin/cupons`
- **AC-10:** `npm run build`, `npm run lint` e `npm run typecheck` passam sem erros

### Query para listagem (admin vê todos)

```typescript
// Usar supabase server client com autenticação admin
const { data: coupons } = await supabase
  .from('coupons')
  .select('*')
  .order('created_at', { ascending: false })
// A policy RLS retorna apenas ativos para não-admin.
// Para admin, is_admin() = true então todos são retornados.
```

### Formatação de desconto na listagem

```typescript
function formatDiscount(type: string, value: number): string {
  if (type === 'percentage') return `${value}%`
  return `R$ ${value.toFixed(2).replace('.', ',')}`
}

function formatMaxUses(current: number, max: number | null): string {
  if (max === null) return `${current} / ∞`
  return `${current} / ${max}`
}
```
