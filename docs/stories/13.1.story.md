## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** uma gestão de cursos completa e robusta no painel admin,
**so that** eu possa criar cursos sem erros, visualizar estatísticas de cada curso, ver quem está matriculado, definir nível e categoria, duplicar cursos e ter uma experiência equivalente a plataformas profissionais de EAD como Tutor LMS.

## Business Value

A gestão atual de cursos tem bug crítico (slug duplicado bloqueia criação), modal do aluno não funciona, e faltam funcionalidades básicas de um LMS profissional: stats por curso, alunos por curso, categorias, nível, duplicação. Isso impede o Fábio de gerenciar sua plataforma com autonomia e eficiência.

## Scope

### IN

#### Bug Fix Obrigatório

- **BUG-1: Slug duplicado** — `src/app/api/admin/courses/route.ts:63` usa `generateSlug(title)` sem verificar unicidade. Se o slug já existe, o INSERT falha com constraint violation. Fix: após gerar o slug, verificar se já existe na tabela e adicionar sufixo `-2`, `-3`, etc. até encontrar um único.

#### Admin Cursos — Melhorias na Listagem

- **Cards de curso** na listagem com: título, thumbnail (se houver), badge de status (Publicado / Rascunho), nível do curso, contagem de módulos/aulas, total de alunos matriculados ativos
- **Filtros na listagem**: busca por título, filtro por status (todos/publicados/rascunhos), ordenação (título A-Z, mais recentes, mais alunos)
- **Botão "Visualizar"** em cada curso: link externo para `/cursos/[slug]` (abre em nova aba)
- **Botão "Duplicar curso"**: copia título (prefixado com "Cópia de "), descrição, preço, nível, categoria — sem módulos/aulas (cópia rasa), slug único automático, status rascunho

#### Painel de Edição do Curso — Novos Campos

- **Nível do curso**: select com opções `iniciante | intermediário | avançado | todos os níveis`
- **Categoria**: campo de texto livre (ex: "Estética Facial", "Massoterapia", "Corporal") — sem tabela separada, salvo como `category` varchar na tabela `courses`
- **Duração total estimada**: calculada automaticamente a partir da soma de `duration_secs` das aulas (exibida como leitura apenas, ex: "2h 34min")

#### Painel do Curso — Aba "Alunos Matriculados"

- Dentro do painel de edição de cada curso, nova aba **"Alunos"** mostrando todos os enrollments ativos/concluídos para aquele curso
- Colunas: avatar + nome, e-mail, data de matrícula, progresso % (baseado em lesson_progress completed / total lições), status (Ativo / Concluído / Cancelado)
- Busca por nome/e-mail dentro da aba
- Botão **"Remover matrícula"** por linha (DELETE enrollment, confirmação via confirm())
- Contador no header da aba: "X alunos matriculados"

#### Painel do Curso — Aba "Estatísticas"

- **Cartões de stats** no topo: Total matriculados | Taxa de conclusão | Progresso médio % | Aulas concluídas (total)
- **Ranking das aulas mais assistidas**: lista top 5 aulas por `lesson_progress` completed count
- Sem gráficos complexos — apenas números e barras simples (mesmo estilo do design system atual)

## Technical Notes

### Fix do Slug Duplicado — courses/route.ts

```typescript
async function generateUniqueSlug(supabase: SupabaseClient, baseSlug: string): Promise<string> {
  let slug = baseSlug
  let suffix = 1
  while (true) {
    const { data } = await supabase.from('courses').select('id').eq('slug', slug).maybeSingle()
    if (!data) return slug
    suffix++
    slug = `${baseSlug}-${suffix}`
  }
}
```

### Migration necessária

```sql
-- Adicionar colunas novas à tabela courses
ALTER TABLE courses ADD COLUMN IF NOT EXISTS level VARCHAR(20) DEFAULT 'todos' CHECK (level IN ('iniciante', 'intermediario', 'avancado', 'todos'));
ALTER TABLE courses ADD COLUMN IF NOT EXISTS category VARCHAR(100);
```

### Progresso % por aluno em um curso

```typescript
// Para cada enrollment, calcular progresso:
// progress% = (aulas com lesson_progress.completed = true) / (total aulas do curso) * 100
const { data: totalLessons } = await supabase
  .from('lessons')
  .select('id, modules!inner(course_id)')
  .eq('modules.course_id', courseId)

const { data: completedLessons } = await supabase
  .from('lesson_progress')
  .select('lesson_id')
  .eq('user_id', userId)
  .eq('completed', true)
  .in(
    'lesson_id',
    totalLessons.map((l) => l.id)
  )

const progress = Math.round((completedLessons.length / totalLessons.length) * 100)
```

### Nova API necessária

- `GET /api/admin/courses/[id]/enrollments` — lista enrollments do curso com progresso por aluno
- `DELETE /api/admin/enrollments/[id]` — remove enrollment (admin only)
- `POST /api/admin/courses/[id]/duplicate` — duplica curso (rasa, sem módulos)

### Estrutura de abas no painel do curso

O `AdminCursosEditor` atual tem apenas uma visão "plana" de edição. Adicionar sistema de tabs:

```
[Conteúdo] [Alunos] [Estatísticas]
```

Tab "Conteúdo" = editor atual (title, desc, price, modules)
Tab "Alunos" = nova lista de enrollments
Tab "Estatísticas" = nova seção de stats
