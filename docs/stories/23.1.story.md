## Story

**As a** administrador da plataforma Fabio Borges,
**I want** configurar o modelo visual do certificado e subir uma assinatura,
**so that** certificados emitidos automaticamente tenham identidade visual aprovada e dados corretos.

## Business Value

O fluxo de conclusão já dispara a emissão do certificado. Sem um builder, a operação depende de um layout hardcoded e não consegue atualizar assinatura, textos ou cores sem alteração de código.

## Scope

### IN

- Criar rota admin `/admin/certificados`.
- Criar template global padrão de certificado.
- Permitir edição de título, texto, emissor, cargo e cor primária.
- Permitir upload de assinatura e assets opcionais de logo/fundo.
- Exibir preview visual do certificado no admin.
- Usar o template salvo na geração de PDF.
- Manter fallback para o modelo atual quando não houver template.
- Corrigir emissão para gravar certificado via service role.

### OUT

- Builder drag-and-drop livre.
- Templates por curso.
- Editor avançado de fontes, camadas e posicionamento.
- Reemissão em massa de certificados já emitidos.

## Acceptance Criteria

- **AC-1:** Admin vê link "Certificados" no menu administrativo.
- **AC-2:** Admin acessa `/admin/certificados` e edita o template padrão.
- **AC-3:** Admin consegue subir assinatura em PNG/JPEG/WebP e salvar a URL no template.
- **AC-4:** Preview reflete textos, cor primária e assinatura configurada.
- **AC-5:** Geração de certificado usa o template salvo quando existir.
- **AC-6:** Se não houver template salvo, a geração mantém fallback funcional.
- **AC-7:** Certificado é inserido em `certificates` via service role após conclusão.
- **AC-8:** `npm run lint`, `npm run typecheck`, `npm test` e `npm run test:e2e` passam.
