## Story

**As a** desenvolvedor do projeto EAD Mentoria Fábio Borges,
**I want** um pipeline CI/CD funcionando via GitHub Actions que faz deploy automático na VPS ao fazer push para `main`,
**so that** qualquer alteração no código chegue à produção de forma segura, rastreável e sem intervenção manual.

## Business Value

Sem CI/CD, cada deploy é manual e arriscado — um comando errado pode derrubar a plataforma no meio de uma venda. Com este pipeline: push para main → lint → build → deploy automático com zero-downtime. O Fábio pode focar no conteúdo enquanto o código vai para produção com segurança.

## Complexity Estimate

## Dev Notes

### Contexto do Projeto

[Source: docs/tech-stack.md — Infraestrutura & DevOps]

Pipeline CI/CD via GitHub Actions. VPS Ubuntu 24.04. PM2 cluster mode. Nginx como reverse proxy com SSL via Let's Encrypt.

### GitHub Actions — Estrutura do Workflow

[Source: docs/prd/8-infraestrutura-vps.md — Pipeline CI/CD]

```
Push main → lint → tsc → build → SSH deploy → git pull → npm ci → build → pm2 reload → health check
```

Secrets necessários no GitHub repo:

- `VPS_HOST` — IP ou hostname da VPS
- `VPS_USER` — usuário SSH (ex: `ubuntu`)
- `VPS_SSH_KEY` — chave SSH privada (RSA ou Ed25519)
- `VPS_PORT` — porta SSH (default: 22)

### PM2 Ecosystem Config

[Source: docs/tech-stack.md — Process Manager PM2]

```javascript
// ecosystem.config.js
module.exports = {
  apps: [
    {
      name: 'ead',
      script: 'node_modules/.bin/next',
      args: 'start',
      instances: 2,
      exec_mode: 'cluster',
      env_production: {
        NODE_ENV: 'production',
        PORT: 3000,
      },
    },
  ],
}
```

### Health Check Route

[Source: docs/prd/8-infraestrutura-vps.md — Monitor UptimeRobot]

UptimeRobot faz ping em `/api/health` a cada 5 minutos. Retorno esperado: HTTP 200 com JSON válido.

### Nginx Config (Reverse Proxy)

[Source: docs/tech-stack.md — Reverse Proxy Nginx 1.24.x]

Porta 3000 (PM2) → Nginx (80/443). SSL termination no Nginx. Certbot gerencia renovação automática do Let's Encrypt.

### Variáveis de Ambiente em Produção

[Source: docs/tech-stack.md — Variáveis de Ambiente Obrigatórias]

Configuradas diretamente no servidor via `.env.production` (não commitado). O GitHub Actions carrega apenas as vars de deploy (SSH). As vars de app ficam na VPS.
