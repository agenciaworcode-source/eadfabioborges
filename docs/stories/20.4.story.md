## Story

**As a** aluno que escolheu parcelar um curso,
**I want** que o checkout Stripe ofereça a opção de parcelamento quando habilitado pelo admin,
**so that** eu possa pagar o curso em parcelas diretamente no Stripe Checkout sem intervenção manual.

## Business Value

Sem a integração no checkout, o parcelamento visível na landing page (20.3) não tem ação — o aluno vê a opção mas não consegue exercer. A ativação de `installments` no Stripe Checkout é simples e de alto impacto: o próprio Stripe lida com a exibição das opções de parcelamento e o split financeiro.

## Complexity Estimate

**T-shirt:** S (Small) — ~2h
**Story Points:** 2

## Scope

### IN

- No provider de Stripe Checkout Session (existente do Epic 16): ao criar Session para compra avulsa de curso, incluir `payment_method_options.card.installments.enabled: true` e `maximum_count: max_installments` quando `max_installments > 1`
- Usar `price` (preço à vista, em centavos) como `unit_amount` base da Session — o Stripe calcula e exibe as parcelas automaticamente no checkout
- Quando `max_installments = 1` (padrão): criar Session sem `installments` (comportamento atual, sem regressão)

### OUT

- Parcelamento para assinaturas de plano (não suportado pelo Stripe Subscriptions — apenas Payment Intents)
- Controle de número de parcelas selecionado pelo aluno (o Stripe exibe as opções; o aluno escolhe)
- Exibição de parcelamento fora do checkout Stripe (já coberto em 20.3)

## Dependencies

- **Story 20.1 deve estar completa** — campos `price_installment` e `max_installments` devem existir
- **Story 20.3 deve estar completa** (recomendado — fluxo coerente)
- Stripe Checkout Session provider existente (Epic 16)
- Verificar se o checkout de cursos avulsos usa `payment_mode: 'payment'` (não `subscription`) — parcelamento no Stripe só funciona com payment mode

## Acceptance Criteria

- **AC-1:** Para um curso com `max_installments = 6`, a Stripe Session criada para checkout desse curso contém `payment_method_options.card.installments.enabled: true` e `maximum_count: 6`
- **AC-2:** Para um curso com `max_installments = 1` (default), a Stripe Session é criada sem a opção `installments` (sem mudança no comportamento atual)
- **AC-3:** O `unit_amount` da Session usa `course.price` (preço à vista em centavos) — não `price_installment`
- **AC-4:** O usuário que visualiza o Stripe Checkout de um curso com parcelamento habilitado vê as opções de parcelamento disponíveis (1x à vista, 2x, ... até `max_installments`)
- **AC-5:** O fluxo de checkout de cursos sem parcelamento não é afetado (sem regressão)

## Technical Notes

### Arquivo a modificar

### Estrutura da Stripe Session com installments

```typescript
// Ao criar a Session para curso com max_installments > 1:
const sessionParams: Stripe.Checkout.SessionCreateParams = {
  mode: 'payment', // OBRIGATÓRIO: installments só funciona com mode: 'payment'
  line_items: [
    {
      price_data: {
        currency: 'brl',
        unit_amount: course.price, // preço à vista em centavos
        product_data: {
          name: course.title,
        },
      },
      quantity: 1,
    },
  ],
  // Adicionar installments apenas se habilitado
  ...(course.max_installments > 1 && {
    payment_method_options: {
      card: {
        installments: {
          enabled: true,
        },
      },
    },
  }),
  // max_installments é passado via payment_intent_data
  ...(course.max_installments > 1 && {
    payment_intent_data: {
      payment_method_options: {
        card: {
          installments: {
            plan: {
              count: course.max_installments,
              interval: 'month',
              type: 'fixed_count',
            },
          },
        },
      },
    },
  }),
}
```

**Nota importante sobre a API do Stripe:** A configuração exata de `installments` no Checkout Session pode variar conforme a versão do Stripe SDK. O dev deve consultar a documentação atual do Stripe para `checkout.sessions.create` com installments. O parâmetro `payment_intent_data.payment_method_options.card.installments` é o canal correto para definir o plano de parcelas.

### Verificação de mode: 'payment'

Parcelamento via Stripe Installments **só funciona com `mode: 'payment'`** (não funciona com `mode: 'subscription'`). Verificar o mode atual do checkout de cursos avulsos — se for diferente, documentar como blocker.
