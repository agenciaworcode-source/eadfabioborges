## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** um endpoint de API que retorne o progresso detalhado de qualquer aluno por curso,
**so that** a tela de drill-down (Story 21.2) tenha dados suficientes para exibir progresso real por curso e por módulo.

## Business Value

Visibilidade do progresso dos alunos é uma das funcionalidades mais pedidas por criadores de conteúdo: permite identificar quem está travado, quais módulos têm maior abandono e quais alunos precisam de suporte. Sem o endpoint, a tela de admin seria apenas uma lista de matrículas sem profundidade — o drill-down de módulo e a última atividade transformam o painel de admin em uma ferramenta de gestão real.

## Scope

### IN

- Endpoint `GET /api/admin/users/[userId]/progress`
- Apenas usuários com role admin podem acessar (401 para não autenticados, 403 para não-admin)
- Response: array de cursos matriculados com:
  - `course_id`, `course_title`, `enrolled_at`, `expires_at`
  - `total_lessons` — total de aulas do curso
  - `completed_lessons` — aulas marcadas como concluídas pelo aluno
  - `progress_percent` — `(completed_lessons / total_lessons * 100)` arredondado para inteiro
  - `last_activity_at` — timestamp da última `lesson_progress` registrada pelo aluno naquele curso
  - `modules[]` — array de módulos com:
    - `module_id`, `module_title`, `module_order`
    - `total_lessons`, `completed_lessons`, `progress_percent`
    - `lessons[]` — array de aulas com `lesson_id`, `lesson_title`, `is_completed`

### OUT

- Tela de admin (Story 21.2)
- Paginação do endpoint (retorna todos os cursos matriculados — quantidade gerenciável)
- Relatório exportável (CSV/PDF)
- Progresso de alunos em plano de assinatura vs curso avulso (retornar todos independente do tipo de acesso)

## Dependencies

- Nenhuma story de Epic 21 como pré-requisito
- Tabelas existentes:
  - `enrollments` — matrículas do aluno em cursos
  - `lesson_progress` — progresso individual por aula
  - `courses`, `modules`, `lessons` — estrutura de conteúdo
  - `users` com campo `role` ou função `is_admin()`

## Acceptance Criteria

- **AC-1:** `GET /api/admin/users/[userId]/progress` retorna `401` para request sem autenticação
- **AC-2:** `GET /api/admin/users/[userId]/progress` retorna `403` para usuário autenticado sem role admin
- **AC-3:** Para um aluno com 2 cursos matriculados, o endpoint retorna array com 2 objetos, cada um com os campos `course_id`, `course_title`, `enrolled_at`, `expires_at`, `total_lessons`, `completed_lessons`, `progress_percent`, `last_activity_at`, `modules`
- **AC-4:** `progress_percent` é calculado corretamente: aluno que completou 3 de 10 aulas → `progress_percent = 30`
- **AC-5:** `last_activity_at` reflete o timestamp mais recente de `lesson_progress` do aluno naquele curso (não a data de matrícula)
- **AC-6:** O array `modules[]` de cada curso contém os módulos na ordem correta (`module_order` ou equivalente), cada um com `total_lessons`, `completed_lessons` e `lessons[]`
- **AC-7:** A query usa os indexes existentes nas tabelas (sem Seq Scan em `lesson_progress` para tabelas grandes) — verificável com EXPLAIN

### Query SQL sugerida (pseudo-código)

```typescript
// 1. Verificar autenticação + role admin
const session = await getSession()
if (!session) return 401
if (!isAdmin(session)) return 403

// 2. Buscar cursos matriculados do aluno
const enrollments = await supabase
  .from('enrollments')
  .select(
    'course_id, enrolled_at, expires_at, courses(id, title, modules(id, title, order, lessons(id, title)))'
  )
  .eq('user_id', userId)

// 3. Para cada curso, buscar lesson_progress do aluno
const progress = await supabase
  .from('lesson_progress')
  .select('lesson_id, completed_at, course_id')
  .eq('user_id', userId)
  .in('course_id', courseIds)
  .order('completed_at', { ascending: false })

// 4. Montar response com cálculos de progresso
```

### Cálculo de progresso

```typescript
const buildProgress = (enrollment: Enrollment, progressRecords: LessonProgress[]) => {
  const courseProgress = progressRecords.filter((p) => p.course_id === enrollment.course_id)
  const completedLessonIds = new Set(courseProgress.map((p) => p.lesson_id))

  const allLessons = enrollment.courses.modules.flatMap((m) => m.lessons)
  const totalLessons = allLessons.length
  const completedLessons = allLessons.filter((l) => completedLessonIds.has(l.id)).length
  const progressPercent = totalLessons > 0 ? Math.round((completedLessons / totalLessons) * 100) : 0
  const lastActivityAt = courseProgress[0]?.completed_at ?? null // já ordenado DESC

  const modules = enrollment.courses.modules.map((module) => {
    const moduleLessons = module.lessons
    const moduleCompleted = moduleLessons.filter((l) => completedLessonIds.has(l.id)).length
    return {
      module_id: module.id,
      module_title: module.title,
      module_order: module.order,
      total_lessons: moduleLessons.length,
      completed_lessons: moduleCompleted,
      progress_percent:
        moduleLessons.length > 0 ? Math.round((moduleCompleted / moduleLessons.length) * 100) : 0,
      lessons: moduleLessons.map((l) => ({
        lesson_id: l.id,
        lesson_title: l.title,
        is_completed: completedLessonIds.has(l.id),
      })),
    }
  })

  return { ...enrollment, totalLessons, completedLessons, progressPercent, lastActivityAt, modules }
}
```
