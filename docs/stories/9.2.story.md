## Story

**As a** administrador da plataforma,
**I want** fazer upload de uma imagem de capa para cada curso,
**so that** o catálogo público e o dashboard do aluno exibam thumbnails reais dos cursos, substituindo os placeholders coloridos e tornando a plataforma visualmente mais atrativa.

## Business Value

Os cards de curso no catálogo público (`/cursos`) e no dashboard do aluno (`/dashboard`) atualmente mostram ícones coloridos gerados por CSS. Um thumbnail real melhora o CTR no catálogo, aumenta percepção de qualidade e reduz a distância visual entre a landing page e a promessa do curso. Esta feature fecha o gap visual mais óbvio antes do go-live.

## Complexity Estimate

**T-shirt:** M (Medium) — ~5h  
**Story Points:** 3

## Scope

### IN

- **`POST /api/admin/courses/[id]/thumbnail`** — nova API route; recebe `multipart/form-data` com campo `file`; faz upload para Supabase Storage bucket `course-thumbnails` (público); atualiza `courses.thumbnail_url`; retorna `{ thumbnailUrl: string }`
- **`AdminCursosEditor.tsx`** — adicionar seção de thumbnail com preview da imagem atual, `<input type="file" accept="image/*">` e botão "Enviar thumbnail"; ao confirmar, chama a API e atualiza o estado local
- **`/cursos` (catálogo)** — quando `thumbnail_url` presente, renderizar `<img>` com `object-fit: cover`; manter `CourseIcon` como fallback quando ausente
- **`/cursos/[slug]` (detalhe)** — quando `thumbnail_url` presente, exibir imagem no hero (coluna da esquerda acima do título); manter layout atual sem imagem como fallback
- **`/dashboard` (student)** — quando `thumbnail_url` presente no curso, usar `<img>` no `.thumb` com `object-fit: cover`; manter `getThumbClass + CourseIcon` como fallback

### OUT

- `next/image` com domínios externos (requer configuração de `remotePatterns` — será avaliado se Storage URL exigir; preferir `<img>` nativo nas páginas públicas para evitar complexidade)
- Redimensionamento / crop da imagem (aceitar qualquer aspect ratio, CSS faz o crop via `object-fit`)
- Limite de tamanho de arquivo no frontend (validar no server: máx 5 MB)
- Remoção / substituição de thumbnail do Storage (arquivo antigo órfão — limpeza futura)
- Thumbnails para módulos ou aulas

## Technical Notes

### API Route — upload para Supabase Storage

```typescript
// src/app/api/admin/courses/[id]/thumbnail/route.ts
import { NextResponse } from 'next/server'
import { createClient } from '@/lib/supabase/server'

const MAX_BYTES = 5 * 1024 * 1024 // 5 MB
const ALLOWED_TYPES = ['image/jpeg', 'image/png', 'image/webp']
const BUCKET = 'course-thumbnails'

export async function POST(request: Request, { params }: { params: { id: string } }) {
  // 1. Auth + admin check
  // 2. Parse formData(), get file
  // 3. Validate size e contentType
  // 4. supabase.storage.from(BUCKET).upload(path, buffer, { contentType, upsert: true })
  // 5. supabase.storage.from(BUCKET).getPublicUrl(path) → publicUrl
  // 6. supabase.from('courses').update({ thumbnail_url: publicUrl }).eq('id', params.id)
  // 7. return { thumbnailUrl: publicUrl }
}
```

> **Bucket público:** usar `getPublicUrl()` (não signed URL) pois thumbnails são públicos. O bucket deve ser criado via migration ou Supabase dashboard com política `allow read for all`.

### Migration para bucket público

Criar `supabase/migrations/20260101000007_storage_course_thumbnails.sql`:

```sql
INSERT INTO storage.buckets (id, name, public)
VALUES ('course-thumbnails', 'course-thumbnails', true)
ON CONFLICT (id) DO NOTHING;

CREATE POLICY "Public read on course-thumbnails"
  ON storage.objects FOR SELECT
  USING (bucket_id = 'course-thumbnails');

CREATE POLICY "Admin upload on course-thumbnails"
  ON storage.objects FOR INSERT
  WITH CHECK (
    bucket_id = 'course-thumbnails'
    AND (SELECT role FROM public.users WHERE id = auth.uid()) = 'admin'
  );

CREATE POLICY "Admin update on course-thumbnails"
  ON storage.objects FOR UPDATE
  USING (
    bucket_id = 'course-thumbnails'
    AND (SELECT role FROM public.users WHERE id = auth.uid()) = 'admin'
  );
```

### AdminCursosEditor — thumbnail section

Adicionar abaixo do campo de preço, no painel de edição do curso selecionado:

```tsx
{
  /* Thumbnail */
}
;<div>
  <label>Thumbnail do curso</label>
  {thumbUrl && (
    <img
      src={thumbUrl}
      alt="thumb"
      style={{
        width: '100%',
        aspectRatio: '16/9',
        objectFit: 'cover',
        borderRadius: '8px',
        marginBottom: '8px',
      }}
    />
  )}
  <input type="file" accept="image/jpeg,image/png,image/webp" onChange={handleThumbChange} />
  {thumbMsg && (
    <p
      style={{
        fontSize: '12px',
        color: thumbMsg.startsWith('Erro') ? 'var(--red)' : 'var(--green)',
      }}
    >
      {thumbMsg}
    </p>
  )}
</div>
```

`thumbUrl` e `thumbMsg` são estados locais. `handleThumbChange` faz `POST /api/admin/courses/{id}/thumbnail` com FormData.

### Exibição no catálogo — fallback pattern

```tsx
// src/app/cursos/page.tsx — dentro do .thumb div
{
  course.thumbnail_url ? (
    <img
      src={course.thumbnail_url}
      alt={course.title}
      style={{ width: '100%', height: '100%', objectFit: 'cover' }}
    />
  ) : (
    <CourseIcon />
  )
}
```

A div `.thumb` já tem `aspect-ratio` e `background` definidos no CSS — a imagem sobrepõe o fundo graciosamente.
