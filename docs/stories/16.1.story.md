## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** que exista uma tabela `plan_courses` associando planos a cursos específicos,
**so that** cada plano possa incluir exatamente os cursos definidos pelo admin — eliminando a dependência do campo `is_vip=true` como proxy de acesso.

## Business Value

Hoje qualquer assinante de plano pago acessa todos os cursos com `is_vip=true`, sem granularidade. Não existe forma de criar um plano que inclua apenas o Curso A e não o Curso B. Essa migration é o pré-requisito zero do Epic 16: sem ela, as stories 16.2, 16.3 e 16.4 não têm onde persistir os dados. A abordagem com seed de retrocompatibilidade garante que todos os assinantes atuais continuam com o mesmo acesso que tinham antes.

## Scope

### IN

#### Tabela `plan_courses`

- Criar tabela `plan_courses` com colunas:
  - `plan_id text NOT NULL REFERENCES public.plans(id) ON DELETE CASCADE`
  - `course_id uuid NOT NULL REFERENCES public.courses(id) ON DELETE CASCADE`
  - `PRIMARY KEY (plan_id, course_id)`
- RLS:
  - SELECT público (qualquer um pode consultar quais cursos pertencem a um plano)
  - INSERT/UPDATE/DELETE apenas admin (`is_admin()`)
- Index de performance: `CREATE INDEX ON plan_courses(plan_id)`

#### Seed de retrocompatibilidade

- Inserir em `plan_courses` todos os cursos com `is_vip = true` associados a todos os planos pagos (`'prata'`, `'ouro'`, `'diamante'`, `'macroempresa'`)
- Formato: produto cartesiano `plans(id) WHERE id != 'free'` × `courses(id) WHERE is_vip = true`
- Usar `INSERT ... ON CONFLICT DO NOTHING` para idempotência

### OUT

- Nenhuma alteração no código da aplicação (fica para 16.2, 16.3, 16.4)
- Remoção ou alteração do campo `is_vip` nas tabelas existentes (mantido para backward compatibility)
- Atualização da função `has_lesson_access()` (fica para 16.3)
- Seed de novos cursos que ainda não existem

## Acceptance Criteria

- **AC-1:** Após aplicar a migration, `\d plan_courses` no psql mostra as colunas `plan_id text` e `course_id uuid` com PK composta `(plan_id, course_id)`
- **AC-2:** FK `plan_id` referencia `plans(id)` com `ON DELETE CASCADE`; FK `course_id` referencia `courses(id)` com `ON DELETE CASCADE`
- **AC-3:** `SELECT * FROM plan_courses WHERE plan_id = 'free'` retorna 0 linhas (plano free não deve ter cursos no seed)
- **AC-4:** `SELECT count(*) FROM plan_courses` retorna N × M onde N = número de planos pagos (4: prata, ouro, diamante, macroempresa) e M = número de cursos com `is_vip = true`
- **AC-5:** A policy RLS permite SELECT para qualquer usuário autenticado ou anônimo; INSERT/DELETE falham para usuário não-admin
- **AC-6:** `supabase db push` (ou aplicação manual) não retorna nenhum erro
- **AC-7:** `EXPLAIN SELECT * FROM plan_courses WHERE plan_id = 'prata'` mostra uso de índice (Index Scan) — não Seq Scan
- **AC-8:** Executar a migration duas vezes (idempotência via `IF NOT EXISTS` e `ON CONFLICT DO NOTHING`) não gera erros nem duplicatas

## Technical Notes

### SQL da migration

```sql
-- Migration: Epic 16.1 — Tabela plan_courses

-- Criar tabela de mapeamento plano → cursos
CREATE TABLE IF NOT EXISTS public.plan_courses (
  plan_id   text NOT NULL REFERENCES public.plans(id) ON DELETE CASCADE,
  course_id uuid NOT NULL REFERENCES public.courses(id) ON DELETE CASCADE,
  PRIMARY KEY (plan_id, course_id)
);

-- RLS
ALTER TABLE public.plan_courses ENABLE ROW LEVEL SECURITY;

CREATE POLICY "plan_courses_read_public"
  ON public.plan_courses FOR SELECT
  USING (true);

CREATE POLICY "plan_courses_write_admin"
  ON public.plan_courses FOR ALL
  USING (is_admin());

-- Index de performance
CREATE INDEX IF NOT EXISTS idx_plan_courses_plan_id ON public.plan_courses(plan_id);

-- Seed de retrocompatibilidade:
-- Associar todos os cursos is_vip=true a todos os planos pagos
INSERT INTO public.plan_courses (plan_id, course_id)
SELECT p.id, c.id
FROM public.plans p
CROSS JOIN public.courses c
WHERE p.id != 'free'
  AND c.is_vip = true
ON CONFLICT DO NOTHING;
```

```sql
SELECT p.id, c.id
FROM public.plans p
CROSS JOIN public.courses c
WHERE p.id != 'free' AND c.is_vip = true
ON CONFLICT DO NOTHING;
-- affected rows = 0
```
