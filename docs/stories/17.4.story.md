## Story

**As a** aluno ou visitante da plataforma,
**I want** inserir um código de cupom na página do curso ou na página de planos antes de comprar,
**so that** eu veja o preço com desconto aplicado antes de iniciar o checkout — sem surpresas no momento do pagamento.

## Business Value

Sem o campo de cupom na UI, toda a infraestrutura de cupons (tabela, admin, validação, checkout) é invisível para o usuário final. Pior: o usuário que tem um cupom e não encontra onde inseri-lo pode desistir da compra. Essa story fecha o ciclo completo do Epic 17, tornando os cupons uma feature utilizável de ponta a ponta — do admin que cria ao aluno que aplica.

## Scope

### IN

#### Página `/cursos/[slug]` — campo de cupom

- Adicionar seção "Tenho um cupom" abaixo do preço do curso (antes do botão de compra)
- Campo text com placeholder "Código do cupom"
- Botão "Aplicar" ao lado do campo
- Ao clicar "Aplicar": chamar `POST /api/checkout/validate-coupon`
- Se cupom válido: exibir preço original riscado + preço com desconto em destaque (verde)
- Se cupom inválido: exibir mensagem de erro em vermelho (usar a mensagem retornada pela API)
- Passar `couponCode` ao iniciar o checkout (para a rota de checkout do curso)

#### Página `/planos` — campo de cupom

- Adicionar campo de cupom similar antes do botão de assinar em cada plano (ou campo global no topo da página)
- Mesma lógica de validação via `POST /api/checkout/validate-coupon` com `itemType: 'plan'`
- Exibir preço do plano com desconto após validação bem-sucedida
- Passar `couponCode` ao iniciar o checkout do plano

#### Estados visuais obrigatórios

- Loading: botão "Aplicar" com spinner enquanto a API responde
- Sucesso: preço original riscado + preço com desconto em verde + ícone de check + texto do desconto (ex: "Desconto de 10% aplicado!")
- Erro: campo com borda vermelha + mensagem de erro da API em vermelho
- Remoção: botão ou link "Remover cupom" que limpa o estado e volta ao preço original

### OUT

- Aplicação automática de cupom via URL (ex: `?cupom=CODIGO` na querystring)
- Campo de cupom no checkout do Stripe (usar apenas na página da plataforma)
- Múltiplos cupons por compra (apenas um cupom por transação)
- Validação offline (sem chamada à API) — sempre validar server-side

## Technical Notes

### Lógica de estado do componente de cupom

```typescript
type CouponState =
  | { status: 'idle' }
  | { status: 'loading' }
  | { status: 'valid'; discountType: string; discountValue: number; discountedPrice: number }
  | { status: 'invalid'; error: string }

// Ao chamar validate-coupon:
const [couponState, setCouponState] = useState<CouponState>({ status: 'idle' })
const [couponCode, setCouponCode] = useState('')

async function handleApplyCoupon() {
  setCouponState({ status: 'loading' })
  const res = await fetch('/api/checkout/validate-coupon', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ code: couponCode, itemType: 'course', itemId: courseId }),
  })
  const data = await res.json()
  if (data.valid) {
    setCouponState({ status: 'valid', ...data })
  } else {
    setCouponState({ status: 'invalid', error: data.error })
  }
}
```

### Exibição de preço com desconto

```tsx
{
  couponState.status === 'valid' && (
    <div className="flex items-center gap-2 mt-2">
      <span className="text-sm text-zinc-400 line-through">{formatPrice(originalPrice)}</span>
      <span className="text-lg font-bold text-green-600">
        {formatPrice(couponState.discountedPrice)}
      </span>
      <span className="text-xs text-green-600">
        {couponState.discountType === 'percentage'
          ? `Desconto de ${couponState.discountValue}% aplicado!`
          : `Desconto de R$ ${couponState.discountValue.toFixed(2).replace('.', ',')} aplicado!`}
      </span>
    </div>
  )
}
```
