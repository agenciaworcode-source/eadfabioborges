## Story

**As a** administrador da plataforma,
**I want** uma infraestrutura centralizada para gerenciar planos, preços e checkout,
**so that** eu possa ajustar valores sem redeploy, integrar qualquer gateway de pagamento sem mudar o código da plataforma, e ter uma única fonte de verdade para regras de acesso.

## Business Value

Atualmente `PLAN_PRICES` está duplicado em 3 arquivos do admin, os preços são hardcoded (exigem deploy para mudar), `subscriptions` tem colunas Stripe-específicas tornando a troca de gateway traumática, e `CheckoutButton` não integra com gateway algum. Esta story fecha esse gap antes de qualquer integração real de pagamento, garantindo que a plataforma possa integrar Stripe, MercadoPago, Hotmart ou qualquer outro gateway apenas trocando variáveis de ambiente — sem alterar o código da plataforma.

## Scope

### IN

- **`src/config/plans.ts`** — single source of truth para metadados de planos (nome, badge, features, ordem de hierarquia, público-alvo, descrição). NÃO inclui preços (ficam no DB).
- **Migration `20260101000008_plans_table.sql`** — tabela `plans` com `id text PK`, `price_monthly integer` (centavos BRL), `price_annual integer`, `sort_order`, `is_active`, `updated_at`; seed dos 4 planos; RLS read público + write admin.
- **`src/lib/checkout/types.ts`** — interface `CheckoutProvider` com `createCourseCheckout()` e `createPlanCheckout()`; tipo `CheckoutSession { url: string }`.
- **`src/lib/checkout/providers/stub.ts`** — provider padrão (desenvolvimento); retorna URL fictícia `{ url: '/planos?checkout=pending' }`.
- **`src/lib/checkout/index.ts`** — factory: lê `CHECKOUT_PROVIDER` env var; exporta `getCheckoutProvider()`.
- **`src/app/api/checkout/plano/route.ts`** — POST `{ planId, billingPeriod }` → chama provider → retorna `{ url }`.
- **`src/app/api/checkout/curso/route.ts`** — POST `{ courseSlug }` → resolve `course.price` do banco → chama provider → retorna `{ url }`.
- **`src/components/checkout/CheckoutButton.tsx`** — FASE 2: troca redirect por POST para `/api/checkout/plano` ou `/api/checkout/curso` → window.location para a URL retornada; spinner de loading.
- **`/admin/planos` (`src/app/(admin)/admin/planos/page.tsx`)** — admin UI: tabela com planos, preços mensal/anual editáveis inline; PATCH `{ price_monthly, price_annual }` via `/api/admin/plans/[id]`.
- **`src/app/api/admin/plans/[id]/route.ts`** — PATCH endpoint; auth + admin check; atualiza `plans.price_monthly` e `plans.price_annual`; retorna plano atualizado.
- **Refatoração** — remover `PLAN_PRICES` hardcoded dos 3 arquivos (`admin/page.tsx`, `admin/alunos/page.tsx`, `admin/relatorios/page.tsx`); substituir por query ao banco (`plans` table) ou import de `src/config/plans.ts` para metadados não-preço.
- **`/planos` page** — exibir preços reais lidos da tabela `plans` (substituir "Preços serão divulgados em breve").

### OUT

## Technical Notes

### `src/config/plans.ts` — metadados (sem preço)

```typescript
export type PlanId = 'free' | 'prata' | 'ouro' | 'diamante' | 'macroempresa'

export interface PlanMeta {
  id: PlanId
  name: string
  badge: string       // CSS class
  audience: string
  description: string
  features: string[]
  billingOptions: ('monthly' | 'annual')[]
  hierarchyLevel: number  // 0=free, 1=prata, 2=ouro, 3=diamante, 4=macroempresa
  isFeatured?: boolean
  isCustomPricing?: boolean
}

export const PLANS: PlanMeta[] = [
  { id: 'free', name: 'Free', badge: '', audience: '', description: '', features: [], billingOptions: [], hierarchyLevel: 0 },
  { id: 'prata', name: 'Prata', badge: 'plan-prata', audience: 'Iniciantes', description: '...', features: [...], billingOptions: ['monthly', 'annual'], hierarchyLevel: 1 },
  { id: 'ouro', name: 'Ouro', badge: 'plan-ouro', audience: 'Profissionais', description: '...', features: [...], billingOptions: ['monthly', 'annual'], hierarchyLevel: 2, isFeatured: true },
  { id: 'diamante', name: 'Diamante', badge: 'plan-diamante', audience: 'Clínicas', description: '...', features: [...], billingOptions: ['monthly', 'annual'], hierarchyLevel: 3 },
  { id: 'macroempresa', name: 'Macroempresa', badge: 'plan-diamante', audience: 'Indústrias / Franquias', description: '...', features: [...], billingOptions: ['annual'], hierarchyLevel: 4, isCustomPricing: true },
]

export const PLAN_MAP = Object.fromEntries(PLANS.map(p => [p.id, p])) as Record<PlanId, PlanMeta>

// Verificar se planA tem acesso ao conteúdo de planB
export function hasAccessToContent(userPlan: PlanId, requiredPlan: PlanId): boolean {
  return (PLAN_MAP[userPlan]?.hierarchyLevel ?? 0) >= (PLAN_MAP[requiredPlan]?.hierarchyLevel ?? 0)
}
```

### Migration — tabela `plans`

```sql
-- supabase/migrations/20260101000008_plans_table.sql
CREATE TABLE public.plans (
  id             text PRIMARY KEY,
  name           text NOT NULL,
  price_monthly  integer NOT NULL DEFAULT 0,  -- BRL centavos (9700 = R$97)
  price_annual   integer NOT NULL DEFAULT 0,
  sort_order     integer NOT NULL DEFAULT 0,
  is_active      boolean NOT NULL DEFAULT true,
  updated_at     timestamptz NOT NULL DEFAULT now()
);

INSERT INTO public.plans (id, name, price_monthly, price_annual, sort_order) VALUES
  ('free',         'Free',         0,     0,      0),
  ('prata',        'Prata',        9700,  97000,  1),
  ('ouro',         'Ouro',         19700, 197000, 2),
  ('diamante',     'Diamante',     39700, 397000, 3),
  ('macroempresa', 'Macroempresa', 0,     0,      4);

-- RLS
ALTER TABLE public.plans ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Public read on plans"
  ON public.plans FOR SELECT USING (true);
CREATE POLICY "Admin write on plans"
  ON public.plans FOR ALL
  USING ((SELECT role FROM public.users WHERE id = auth.uid()) = 'admin');
```

> Preços em centavos para evitar ponto flutuante. Helper `formatPrice(cents)` em `plans.ts`.

### Gateway Adapter Pattern

```typescript
// src/lib/checkout/types.ts
export interface CheckoutSession {
  url: string
}

export interface CreateCourseCheckoutParams {
  courseId: string
  courseTitle: string
  userId: string
  userEmail: string
  priceAmountCents: number
  successUrl: string
  cancelUrl: string
}

export interface CreatePlanCheckoutParams {
  planId: string
  planName: string
  userId: string
  userEmail: string
  priceAmountCents: number
  billingPeriod: 'monthly' | 'annual'
  successUrl: string
  cancelUrl: string
}

export interface CheckoutProvider {
  createCourseCheckout(params: CreateCourseCheckoutParams): Promise<CheckoutSession>
  createPlanCheckout(params: CreatePlanCheckoutParams): Promise<CheckoutSession>
}
```

```typescript
// src/lib/checkout/index.ts
import type { CheckoutProvider } from './types'

export function getCheckoutProvider(): CheckoutProvider {
  const provider = process.env.CHECKOUT_PROVIDER ?? 'stub'
  switch (provider) {
    case 'stub':
      return import('./providers/stub').then((m) => m.stubProvider) as never
    // case 'stripe': return stripeProvider
    // case 'mercadopago': return mpProvider
    default:
      throw new Error(`Unknown CHECKOUT_PROVIDER: ${provider}`)
  }
}
```

> Para adicionar Stripe: criar `src/lib/checkout/providers/stripe.ts` implementando `CheckoutProvider`, adicionar `case 'stripe'` no factory, setar `CHECKOUT_PROVIDER=stripe`. Zero mudança na plataforma.
