## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** criar, editar, reordenar e deletar módulos e aulas dentro de cada curso pelo painel admin,
**so that** eu possa montar o currículo completo dos cursos sem precisar acessar o banco de dados.

## Business Value

Atualmente o admin só permite editar título, descrição e preço de um curso. Não existe forma de criar conteúdo — módulos e aulas — pela interface. Isso significa que a plataforma não tem conteúdo assistível e os alunos não conseguem usar o produto. Este é o desbloqueador central de todo o Epic 6.

## Scope

### IN

- **Expandir `AdminCursosEditor`**: ao selecionar um curso, exibir seção "Conteúdo do curso" com lista completa de módulos e aulas
- **CRUD de módulos** dentro do editor lateral:
  - Criar módulo (campo: título)
  - Editar título do módulo inline
  - Deletar módulo (com confirmação, cascade apaga aulas)
  - Reordenar módulos com botões ↑ ↓
  - Toggle `is_free_preview` no módulo
- **CRUD de aulas** dentro de cada módulo (expandido/colapsado):
  - Criar aula (campos: título, vimeo_id, duration_secs, is_free_preview)
  - Editar aula inline
  - Deletar aula (com confirmação)
  - Reordenar aulas com botões ↑ ↓
- **APIs admin:**
  - `POST /api/admin/courses/[courseId]/modules` — criar módulo
  - `PATCH /api/admin/modules/[id]` — atualizar módulo (title, order, is_free_preview)
  - `DELETE /api/admin/modules/[id]` — deletar módulo
  - `POST /api/admin/modules/[moduleId]/lessons` — criar aula
  - `PATCH /api/admin/lessons/[id]` — atualizar aula (title, vimeo_id, duration_secs, order, is_free_preview)
  - `DELETE /api/admin/lessons/[id]` — deletar aula
- Todas as APIs verificam `role = 'admin'` via Supabase Auth
- Feedback visual: loading states, mensagens de sucesso/erro
- Atualização otimista do estado local após cada operação

### OUT

- Upload de thumbnail de curso
- Drag-and-drop de reordenação (botões ↑ ↓ suficiente para MVP)
- Preview de vídeo Vimeo no editor
- Quiz builder (Story 6.2)
- Assignment builder (Story 6.3)
- Bulk operations (deletar múltiplos)

## Technical Notes

### Padrão de verificação admin nas APIs

```typescript
// Padrão usado em todas as routes admin (baseado em Story 5.1)
import { createClient } from '@/lib/supabase/server'

const supabase = createClient()
const {
  data: { user },
} = await supabase.auth.getUser()
if (!user) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })

const { data: profile } = await supabase.from('users').select('role').eq('id', user.id).single()
if (profile?.role !== 'admin') return NextResponse.json({ error: 'Forbidden' }, { status: 403 })
```

### Estrutura de dados esperada

```typescript
// Módulo
interface Module {
  id: string
  course_id: string
  title: string
  order: number
  is_free_preview: boolean
  lessons: Lesson[]
}

// Aula
interface Lesson {
  id: string
  module_id: string
  title: string
  vimeo_id: string | null
  duration_secs: number
  order: number
  is_free_preview: boolean
}
```

### Query para carregar módulos+aulas do curso

```typescript
const { data } = await supabase
  .from('modules')
  .select(
    'id, title, order, is_free_preview, lessons(id, title, vimeo_id, duration_secs, order, is_free_preview)'
  )
  .eq('course_id', courseId)
  .order('order')
```

### Padrão de reordenação

Ao clicar ↑ ou ↓: trocar o campo `order` entre o item e seu vizinho. Fazer PATCH de ambos os itens em paralelo.
