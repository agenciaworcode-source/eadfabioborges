## Story

**As a** aluno da plataforma,
**I want** ver claramente quando meu acesso a um curso expira e receber uma mensagem explicativa quando ele já expirou,
**so that** eu saiba exatamente qual é minha situação de acesso — e para que a tentativa de gerar um certificado em curso sem certificado retorne um erro claro ao invés de falhar silenciosamente.

## Business Value

Sem esse frontend, a lógica implementada nas Stories 15.2 e 15.3 é invisível para o aluno: ele veria conteúdo bloqueado sem entender o motivo (confundindo expiração com falta de matrícula), não saberia até quando tem acesso, e poderia tentar emitir um certificado de um curso que não emite — gerando confusão e tickets de suporte. Essa story fecha o ciclo do Epic 15 tornando a experiência do aluno transparente e autoexplicativa.

## Scope

### IN

#### AccessGate — estado de expiração

- Modificar `src/components/shared/AccessGate.tsx` para receber e verificar `enrollment.expires_at`
- Se `expires_at !== null AND expires_at < now()`: renderizar bloqueio com mensagem específica:
  > "Seu acesso a este curso expirou em [DD/MM/AAAA]."
  > Link "Renovar acesso" → `/cursos/[slug]` (compra avulso)
- Se `hasAccess = false` sem expiração: manter comportamento atual (mensagem de "Conteúdo bloqueado")
- Se `hasAccess = true`: renderizar conteúdo normalmente

#### Hook `useEnrollment` — expor expires_at

- Modificar `useEnrollment` para retornar `expiresAt: string | null` além de `hasAccess`
- Buscar `expires_at` do enrollment do usuário no curso

#### Dashboard do aluno — badge de validade

- Localizar a tela de listagem de cursos do aluno (`/dashboard/cursos` ou equivalente)
- Para cursos com `expires_at !== null`: exibir badge/texto "Acesso válido até DD/MM/AAAA" abaixo do título do curso
- Para cursos com `expires_at = null`: não exibir data (sem alteração visual)
- Para cursos com `expires_at < now()`: exibir badge "Acesso expirado" em vermelho/destaque

#### API de certificado — verificar certificate_enabled

- Modificar `src/app/api/certificate/generate/route.ts`
- Antes de gerar o PDF: buscar `courses.certificate_enabled` para o curso em questão
- Se `certificate_enabled = false`: retornar `{ error: 'Este curso não emite certificado' }` com status 403
- Se `certificate_enabled = true`: prosseguir com geração normalmente

### OUT

- Notificação por e-mail de aviso de expiração próxima
- Botão de renovação de acesso com fluxo de re-compra completo (apenas link para página do curso)
- Alterações na tela de emissão de certificado além da validação de API
- Testes automatizados de componentes (foco nos ACs manuais)

## Acceptance Criteria

- **AC-1:** Um aluno com enrollment `status = 'expired'` (ou `expires_at < now()`) que acessa uma aula do curso vê a mensagem "Seu acesso a este curso expirou em [data formatada DD/MM/AAAA]" no lugar do conteúdo
- **AC-2:** A data exibida na mensagem de expiração corresponde ao valor de `expires_at` do enrollment, formatada como DD/MM/AAAA (ex: "15/07/2026")
- **AC-3:** Um aluno com `expires_at = null` (vitalício) que acessa uma aula do curso não vê nenhuma mensagem de expiração — o conteúdo é exibido normalmente
- **AC-4:** No dashboard do aluno, cursos com `expires_at !== null` e `expires_at > now()` exibem o texto "Acesso válido até DD/MM/AAAA"
- **AC-5:** No dashboard do aluno, cursos com `expires_at = null` não exibem data de validade
- **AC-6:** No dashboard do aluno, cursos com `expires_at < now()` exibem badge/texto "Acesso expirado" em destaque visual diferente dos cursos ativos
- **AC-7:** Uma chamada `POST /api/certificate/generate` para um curso com `certificate_enabled = false` retorna status 403 e body `{ error: 'Este curso não emite certificado' }`
- **AC-8:** Uma chamada `POST /api/certificate/generate` para um curso com `certificate_enabled = true` e aluno com acesso válido retorna status 200 e gera o PDF normalmente
- **AC-9:** `npm run build`, `npm run lint` e `npm run typecheck` passam sem erros

### Atualização do hook useEnrollment

```typescript

// Exemplo de interface de retorno atualizada:
interface UseEnrollmentReturn {
  hasAccess: boolean
  isLoading: boolean

  isExpired: boolean // expires_at !== null && new Date(expires_at) < new Date()
```

### Atualização do AccessGate

```typescript
// Novo estado a tratar:
// 1. isLoading → skeleton
// 2. hasAccess → children
// 3. isExpired → mensagem de expiração com data formatada
// 4. !hasAccess && !isExpired → bloqueio atual (plano/compra)

// Formatação da data de expiração
function formatExpiry(isoDate: string): string {
  const d = new Date(isoDate)
  return d.toLocaleDateString('pt-BR', { day: '2-digit', month: '2-digit', year: 'numeric' })
}
```
