## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** que o banco de dados suporte todos os novos campos do Course Builder Avançado,
**so that** as stories 22.2, 22.3, 22.4 e 22.5 possam ser implementadas sem novas migrations.

## Business Value

Esta migration é o pré-requisito zero de todo o Epic 22. Sem ela, nenhum dos novos campos (intro vídeo, bullets de aprendizagem, YouTube, timer de quiz, etc.) tem onde ser persistido. A abordagem consolidada em uma única migration evita múltiplos arquivos SQL e garante que todas as stories seguintes possam rodar em paralelo imediatamente após esta estar concluída.

## Scope

### IN

**Tabela `courses` — 7 novos campos:**

- `intro_video_url text` — URL do vídeo de apresentação (YouTube ou Vimeo)
- `what_you_learn text[]` — Lista de bullets "o que você aprenderá" (array de strings)
- `requirements text[]` — Lista de pré-requisitos (array de strings)
- `target_audience text` — Descrição do público-alvo
- `tags text[]` — Tags para categorização e SEO
- `max_students integer DEFAULT 0` — Máx de alunos (0 = ilimitado)
- `slug text UNIQUE` — Slug editável (coluna já pode existir; ALTER apenas se não existir)

**Tabela `modules` — 1 novo campo:**

- `description text` — Sumário/descrição do módulo (opcional)

**Tabela `lessons` — 3 novos campos:**

- `youtube_url text` — URL do vídeo YouTube (alternativa ao vimeo_id)
- `video_thumbnail_url text` — Thumbnail customizado do player de vídeo
- `completion_percent integer DEFAULT 0` — % do vídeo necessário para marcar concluído (0 = sem requisito)

**Tabela `quizzes` — 4 novos campos:**

- `time_limit_secs integer DEFAULT 0` — Tempo limite em segundos (0 = sem limite)
- `feedback_mode text DEFAULT 'after_all'` — Modo de feedback: after_all / after_each / unlimited_retry
- `randomize_questions boolean DEFAULT false` — Randomizar ordem das questões
- `max_questions_shown integer DEFAULT 0` — Limite de questões exibidas por tentativa (0 = todas)

**Tabela `questions` — novo campo:**

- `question_order integer DEFAULT 0` — Ordem da questão dentro do quiz (para drag-and-drop de questões)

**Enum `question_type` (ou coluna `type text`):**

- Verificar se `type` é enum ou text; se for enum, adicionar valores: `fill_in_blanks`, `short_answer`, `matching`, `ordering`
- Se for `text`, nenhuma alteração de enum necessária (valores já são livres)

**Tabela `lesson_progress` — 1 novo campo:**

- `resume_position_secs integer DEFAULT 0` — Posição de auto-resume do vídeo

### OUT

- Nenhuma alteração em RLS (novas colunas herdam políticas existentes das tabelas)
- Nenhuma alteração em índices além do UNIQUE em `slug` (se não existir)
- Nenhuma alteração no código da aplicação (fica para 22.2–22.5)
- Seed de dados para os novos campos

## Dependencies

### SQL da migration

```sql
-- ============================================================
-- Epic 22 — Course Builder Avançado
-- Migration: 20260619000001_course_builder_advanced.sql
-- ============================================================

-- TABELA: courses — 7 novos campos
ALTER TABLE public.courses
  ADD COLUMN IF NOT EXISTS intro_video_url    text,
  ADD COLUMN IF NOT EXISTS what_you_learn     text[] DEFAULT '{}',
  ADD COLUMN IF NOT EXISTS requirements       text[] DEFAULT '{}',
  ADD COLUMN IF NOT EXISTS target_audience    text,
  ADD COLUMN IF NOT EXISTS tags               text[] DEFAULT '{}',
  ADD COLUMN IF NOT EXISTS max_students       integer NOT NULL DEFAULT 0,
  ADD COLUMN IF NOT EXISTS slug               text;

-- Garantir unicidade do slug (apenas se não existir o constraint)
DO $$
BEGIN
  IF NOT EXISTS (
    SELECT 1 FROM pg_constraint
    WHERE conname = 'courses_slug_key'
      AND conrelid = 'public.courses'::regclass
  ) THEN
    ALTER TABLE public.courses ADD CONSTRAINT courses_slug_key UNIQUE (slug);
  END IF;
END $$;

-- TABELA: modules — 1 novo campo
ALTER TABLE public.modules
  ADD COLUMN IF NOT EXISTS description text;

-- TABELA: lessons — 3 novos campos
ALTER TABLE public.lessons
  ADD COLUMN IF NOT EXISTS youtube_url          text,
  ADD COLUMN IF NOT EXISTS video_thumbnail_url  text,
  ADD COLUMN IF NOT EXISTS completion_percent   integer NOT NULL DEFAULT 0;

-- TABELA: quizzes — 4 novos campos
ALTER TABLE public.quizzes
  ADD COLUMN IF NOT EXISTS time_limit_secs      integer NOT NULL DEFAULT 0,
  ADD COLUMN IF NOT EXISTS feedback_mode        text    NOT NULL DEFAULT 'after_all',
  ADD COLUMN IF NOT EXISTS randomize_questions  boolean NOT NULL DEFAULT false,
  ADD COLUMN IF NOT EXISTS max_questions_shown  integer NOT NULL DEFAULT 0;

-- TABELA: questions — 1 novo campo para ordering
ALTER TABLE public.questions
  ADD COLUMN IF NOT EXISTS question_order integer NOT NULL DEFAULT 0;

-- TABELA: lesson_progress — auto-resume position
ALTER TABLE public.lesson_progress
  ADD COLUMN IF NOT EXISTS resume_position_secs integer NOT NULL DEFAULT 0;
```

### Verificação de enum em `questions.type`

Antes de criar a migration, verificar com:

```sql
SELECT data_type FROM information_schema.columns
WHERE table_name = 'questions' AND column_name = 'type';
```

- Se retornar `text`: nenhuma alteração de enum necessária. Novos tipos (`fill_in_blanks`, `short_answer`, `matching`, `ordering`) podem ser inseridos diretamente.
- Se retornar `USER-DEFINED`: adicionar valores ao enum antes do ALTER TABLE.

### Padrão TypeScript para arrays Supabase

```typescript
// Em database.ts, para campos text[]:
what_you_learn: string[] | null
requirements: string[] | null
tags: string[] | null

// Padrão de fix para never em queries com novos campos:
const course = data as unknown as CourseRow
```

### Checagem de migrations existentes

Antes de criar a migration, verificar os arquivos existentes em `supabase/migrations/` para garantir que os campos ainda não foram adicionados por outra migration do Epic 14.x ou 15.x.
