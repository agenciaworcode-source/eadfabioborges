## Story

**As a** aluno da plataforma,
**I want** que meu acesso às aulas seja controlado pelos cursos que estão realmente mapeados no meu plano,
**so that** planos com diferentes composições de cursos resultem em acessos distintos — sem que todos os assinantes vejam os mesmos cursos independentemente do plano contratado.

## Business Value

A função `has_lesson_access()` atual usa `courses.is_vip = true` como critério de acesso via plano — qualquer assinante ativo acessa qualquer curso marcado como VIP, sem distinção de plano. Com a tabela `plan_courses` criada na 16.1, é possível ter controle granular: Prata acessa cursos A e B; Ouro acessa A, B e C. Essa story é o coração do Epic 16: sem ela, toda a UI e a nova tabela são inoperantes do ponto de vista de controle de acesso real.

## Technical Notes

### Mapeamento subscription → plan_id

A tabela `subscriptions` tem coluna `plan subscription_plan` (enum: `'monthly'`, `'annual'`, `'lifetime'`) que indica o período, não o `plan_id`. O `plan_id` real (ex: `'prata'`, `'ouro'`) está em `users.plan` ou precisa ser derivado de outro campo.

**[AUTO-DECISION]** Como mapear subscription → plan_id: verificar se existe coluna `plan_id text` ou similar em `subscriptions` que não aparece na migration 3. Se não existir, usar `users.plan` como proxy (campo que indica o plano do usuário). Caso `users.plan` também não exista, o mapeamento deve ser resolvido pelo @dev via análise do schema atual antes de escrever a função.

```sql
-- Novo branch de subscription na função has_lesson_access():
OR EXISTS (
  SELECT 1
  FROM public.lessons l
  JOIN public.modules m ON m.id = l.module_id
  JOIN public.subscriptions s ON s.user_id = auth.uid()
  JOIN public.plan_courses pc ON pc.course_id = m.course_id
  -- mapear s para plan_id: ajustar conforme schema atual
  -- opção A: se subscriptions tiver plan_id text
  --   AND pc.plan_id = s.plan_id
  -- opção B: se users.plan contiver o plan_id
  --   JOIN public.users u ON u.id = auth.uid() AND pc.plan_id = u.plan
  WHERE l.id = p_lesson_id
    AND s.status = 'active'
    AND s.period_end > now()
)
```
