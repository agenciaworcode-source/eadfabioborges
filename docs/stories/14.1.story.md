## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** que a tabela `lessons` suporte múltiplos tipos de conteúdo (vídeo, texto, PDF, embed externo),
**so that** eu possa criar aulas com formatos além do Vimeo — sem quebrar as aulas de vídeo já existentes.

## Business Value

Atualmente toda aula exige um `vimeo_id`. Isso bloqueia a criação de aulas baseadas em texto, PDF ou embeds externos. A migração dessa story é o pré-requisito técnico zero: sem ela as stories 14.2 (admin) e 14.3 (player) não têm onde salvar os dados. A abordagem backward-compatible garante que os cursos publicados continuam funcionando durante e após o rollout.

**T-shirt:** XS (Extra Small) — ~1h

## Scope

### IN

- Criar enum PostgreSQL `lesson_type` com valores: `video`, `text`, `pdf`, `embed`
- Adicionar coluna `type lesson_type NOT NULL DEFAULT 'video'` na tabela `lessons`
- Adicionar coluna `content_body text` (conteúdo HTML/markdown para aulas de texto)
- Adicionar coluna `embed_url text` (URL de embed externo)
- Adicionar coluna `pdf_url text` (URL do arquivo PDF — Supabase Storage ou qualquer CDN)
- Manter coluna `vimeo_id` intacta (backward compatible — nenhum dado existente é afetado)
- RLS: nenhuma mudança necessária (novas colunas herdam as políticas existentes da tabela `lessons`)

### OUT

- Nenhuma alteração no código da aplicação (fica para 14.2 e 14.3)
- Seed de dados de exemplo para cada tipo
- Bucket de Supabase Storage para PDFs (pode ser criado separadamente)
- Validações de negócio no banco (CHECK constraints por tipo) — a validação fica na camada de API

### SQL da migration

```sql
-- Criar enum de tipos de aula
CREATE TYPE public.lesson_type AS ENUM ('video', 'text', 'pdf', 'embed');

-- Adicionar novas colunas à tabela lessons
ALTER TABLE public.lessons
  ADD COLUMN IF NOT EXISTS type         public.lesson_type NOT NULL DEFAULT 'video',
  ADD COLUMN IF NOT EXISTS content_body text,
  ADD COLUMN IF NOT EXISTS embed_url    text,
  ADD COLUMN IF NOT EXISTS pdf_url      text;
```
