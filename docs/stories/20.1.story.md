## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** que a tabela `courses` suporte preço parcelado total e número máximo de parcelas,
**so that** as stories 20.2, 20.3 e 20.4 tenham onde persistir e consultar a precificação avançada de cursos avulsos.

## Business Value

Parcelamento é uma das principais alavancas de conversão no Brasil — cursos com ticket acima de R$200 têm conversão significativamente maior quando oferecidos em parcelas. Sem os campos no banco, nenhuma das outras stories do Epic 20 pode ser implementada. A migration garante backward compatibility: `max_installments DEFAULT 1` significa que todos os cursos existentes mantêm comportamento atual (à vista apenas).

## Complexity Estimate

**T-shirt:** XS (Extra Small) — ~1h
**Story Points:** 1

## Scope

### IN

- `ALTER TABLE public.courses ADD COLUMN IF NOT EXISTS price_installment integer` (NULL = sem precificação parcelada; quando preenchido, representa o preço total em centavos para parcelamento)
- `ALTER TABLE public.courses ADD COLUMN IF NOT EXISTS max_installments integer NOT NULL DEFAULT 1` (1 = apenas à vista; > 1 = parcelamento habilitado)
- Sem seed necessário — `price_installment` NULL e `max_installments = 1` são o default correto para todos os cursos existentes

### OUT

- Lógica de validação `price_installment >= price` (validação na camada de API — Story 20.2)
- Campos de Stripe para installments (Story 20.4)
- Formulário admin (Story 20.2)
- Exibição no frontend (Story 20.3)

## Dependencies

- Nenhuma story de Epic 20 como pré-requisito
- Nenhuma dependência de outros epics
- Schema atual de `courses` — verificar migration base para confirmar nome da coluna de preço (`price` em centavos)

## Acceptance Criteria

- **AC-1:** Após aplicar a migration, `\d courses` mostra a coluna `price_installment integer` (nullable)
- **AC-2:** Após aplicar a migration, `\d courses` mostra a coluna `max_installments integer NOT NULL DEFAULT 1`
- **AC-3:** Todos os cursos existentes têm `max_installments = 1` após a migration (default aplicado)
- **AC-4:** Todos os cursos existentes têm `price_installment IS NULL` após a migration (sem parcelamento por padrão)
- **AC-5:** `supabase db push` não retorna nenhum erro
- **AC-6:** Executar a migration duas vezes (idempotência via `ADD COLUMN IF NOT EXISTS`) não gera erros

## Technical Notes

### SQL da migration

```sql
-- Migration: Epic 20.1 — Campos de Precificação Avançada

-- Preço total parcelado em centavos (NULL = sem parcelamento)
ALTER TABLE public.courses
  ADD COLUMN IF NOT EXISTS price_installment integer;

-- Número máximo de parcelas (1 = à vista apenas, padrão)
ALTER TABLE public.courses
  ADD COLUMN IF NOT EXISTS max_installments integer NOT NULL DEFAULT 1;
```

### Verificação pós-aplicação

```sql
-- Confirmar colunas adicionadas
SELECT column_name, data_type, column_default, is_nullable
FROM information_schema.columns
WHERE table_name = 'courses'
  AND column_name IN ('price_installment', 'max_installments')
ORDER BY ordinal_position;

-- Confirmar defaults nos registros existentes
SELECT id, title, price, price_installment, max_installments
FROM courses
LIMIT 5;
-- price_installment: null | max_installments: 1
```
