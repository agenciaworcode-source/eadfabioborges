## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** que o sistema calcule automaticamente a data de expiração da matrícula no momento do enrollment e revogue periodicamente os acessos vencidos,
**so that** cursos com prazo limitado sejam efetivamente bloqueados quando o tempo de acesso do aluno expirar — sem intervenção manual.

## Business Value

Sem essa lógica, o campo `access_days` da Story 15.2 não tem efeito prático: o aluno com prazo de 30 dias continuaria com acesso vitalício porque `expires_at` nunca seria preenchido e `has_lesson_access()` não verificaria expiração. Essa story fecha o ciclo: calcula `expires_at` no momento da matrícula, mantém `enrollment_status = 'expired'` para acessos vencidos via cron, e atualiza a função RLS para honrar a expiração.

## Scope

### IN

#### Migration complementar — enum `enrollment_status`

- Adicionar valor `'expired'` ao enum `enrollment_status` existente
- `ALTER TYPE enrollment_status ADD VALUE IF NOT EXISTS 'expired';`
- Arquivo de migration separado da 15.1 para manter responsabilidades separadas

#### Cálculo de `expires_at` na matrícula manual (API admin enroll)

- Em `src/app/api/admin/enroll/route.ts`: após inserir o enrollment, buscar `courses.access_days`
- Se `access_days > 0`: `expires_at = enrolled_at + interval '${access_days} days'`
- Se `access_days IS NULL`: `expires_at = NULL` (vitalício)
- Atualizar o enrollment recém-criado com o `expires_at` calculado

#### Cálculo de `expires_at` no checkout (webhook Stripe)

- Verificar o handler de webhook Stripe que confirma pagamento e cria enrollment
- Aplicar a mesma lógica de cálculo de `expires_at` baseada em `courses.access_days`

#### Cron job — revogação de acessos expirados

- Criar `src/app/api/cron/expire-enrollments/route.ts`
- Autenticação via header `Authorization: Bearer {CRON_SECRET}` (mesmo padrão do cron existente)
- Query de revogação:
  ```sql
  UPDATE enrollments
  SET status = 'expired'
  WHERE expires_at < now()
    AND status = 'active'
  ```
- Registrar no `vercel.json` com schedule diário: `"0 2 * * *"` (2h UTC = madrugada horário Brasil)
- Response: `{ updated: N, timestamp: ISO }` para monitoramento

#### Atualização da função `has_lesson_access()` no Supabase

- Criar migration que recria a função com verificação adicional de expiração na branch de enrollment:
  ```sql
  AND e.status = 'active'
  AND (e.expires_at IS NULL OR e.expires_at > now())
  ```
- A branch de assinatura ativa (`subscriptions`) não é afetada (assinaturas têm `period_end`)
- A branch de free preview não é afetada

### OUT

- Interface frontend de exibição de expiração (fica para Story 15.4)
- Notificação por e-mail de aviso de expiração próxima
- Renovação de acesso via interface do aluno
- Reativar enrollment expirado (fluxo de re-compra — futuro)
- Expiração de assinaturas (gerenciada por `period_end` existente)

## Acceptance Criteria

- **AC-1:** Ao criar uma matrícula manual via `POST /api/admin/enroll` para um curso com `access_days = 30`, o enrollment persistido tem `expires_at = enrolled_at + 30 days` (tolerância de ±1 minuto)
- **AC-2:** Ao criar uma matrícula manual para um curso com `access_days = null`, o enrollment persistido tem `expires_at = null`
- **AC-3:** O cron `GET /api/cron/expire-enrollments` com header `Authorization: Bearer {CRON_SECRET}` retorna status 200 e body `{ updated: N, timestamp: "..." }`
- **AC-4:** Após executar o cron, enrollments com `expires_at < now() AND status = 'active'` passam a ter `status = 'expired'`; enrollments com `expires_at > now()` ou `expires_at = null` permanecem `'active'`
- **AC-5:** Um aluno com enrollment `status = 'expired'` não consegue carregar aulas do curso via `has_lesson_access()` — a função retorna `false` para esse aluno nesse curso
- **AC-6:** Um aluno com enrollment `status = 'active'` e `expires_at = null` continua com acesso normalmente após a atualização da função RLS
- **AC-7:** Um aluno com enrollment `status = 'active'` e `expires_at > now()` continua com acesso normalmente
- **AC-8:** `npm run build`, `npm run lint` e `npm run typecheck` passam sem erros

### Migration — atualizar has_lesson_access()

```sql
-- Migration: Epic 15 — has_lesson_access com verificação de expiração
CREATE OR REPLACE FUNCTION public.has_lesson_access(p_lesson_id uuid)
RETURNS boolean AS $$
  SELECT EXISTS (
    -- Access via course enrollment (ativo e não expirado)
    SELECT 1
    FROM public.lessons l
    JOIN public.modules m ON m.id = l.module_id
    JOIN public.enrollments e ON e.course_id = m.course_id
    WHERE l.id = p_lesson_id
      AND e.user_id = auth.uid()
      AND e.status = 'active'
      AND (e.expires_at IS NULL OR e.expires_at > now())
  )
  OR EXISTS (
    -- Access via active subscription (VIP plan covers vip courses)
    SELECT 1
    FROM public.lessons l
    JOIN public.modules m ON m.id = l.module_id
    JOIN public.courses c ON c.id = m.course_id
    JOIN public.subscriptions s ON s.user_id = auth.uid()
    WHERE l.id = p_lesson_id
      AND c.is_vip = true
      AND s.status = 'active'
      AND s.period_end > now()
  )
  OR EXISTS (
    -- Free preview lessons are always accessible
    SELECT 1
    FROM public.lessons l
    JOIN public.modules m ON m.id = l.module_id
    WHERE l.id = p_lesson_id
      AND m.is_free_preview = true
  );
$$ LANGUAGE sql SECURITY DEFINER STABLE;
```

Arquivo: `supabase/migrations/20260617000004_epic15_has_lesson_access_expiry.sql`

### Cálculo de expires_at no enroll route

```typescript
// Após inserir o enrollment, buscar access_days do curso
const { data: course } = await supabase
  .from('courses')
  .select('access_days')
  .eq('id', courseId)
  .single()

const accessDays = (course as { access_days: number | null } | null)?.access_days ?? null

if (accessDays && accessDays > 0) {
  const expiresAt = new Date()
  expiresAt.setDate(expiresAt.getDate() + accessDays)
  await supabase
    .from('enrollments')
    .update({ expires_at: expiresAt.toISOString() })
    .eq('id', enrollmentId)
}
// Se accessDays é null: expires_at permanece null (vitalício)
```

### Cron — expire-enrollments

```typescript
// src/app/api/cron/expire-enrollments/route.ts
import { NextResponse } from 'next/server'
import { createServiceClient } from '@/lib/supabase/service'

export async function GET(request: Request) {
  const secret = request.headers.get('authorization')?.replace('Bearer ', '')
  if (!secret || secret !== process.env.CRON_SECRET) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
  }

  const supabase = createServiceClient()

  const { data, error } = await supabase
    .from('enrollments')
    .update({ status: 'expired' })
    .lt('expires_at', new Date().toISOString())
    .eq('status', 'active')
    .select('id')

  if (error) {
    console.error('[cron/expire-enrollments] error:', error)
    return NextResponse.json({ error: error.message }, { status: 500 })
  }

  return NextResponse.json({
    updated: data?.length ?? 0,
    timestamp: new Date().toISOString(),
  })
}
```
