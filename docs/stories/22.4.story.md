## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** suporte nativo a YouTube nas aulas, thumbnail customizado no player, requisito de % assistido para marcar concluído e retomada automática de onde o aluno parou,
**so that** a experiência de vídeo da plataforma seja comparável ao Tutor LMS e ao próprio YouTube.

## Business Value

Atualmente 100% dos vídeos exigem Vimeo por ID — sem YouTube, sem fallback. O Tutor LMS suporta YouTube, Vimeo e self-hosted nativamente. O auto-resume elimina a frustração de alunos que fecham o browser e voltam sem saber onde pararam. O completion requirement impede que alunos marquem aulas como concluídas sem realmente assistir.

## Scope

### IN

**Admin — `LessonEditor.tsx` (aulas do tipo `video`):**

- Campo "Vídeo YouTube" (URL): input text `youtube_url` — NOVO
- Campo "Thumbnail do player" (URL): input text `video_thumbnail_url` — NOVO
- Campo "% assistido para concluir" (número 0–100, 0 = sem requisito): `completion_percent` — NOVO
- Lógica de exibição: mostrar campo `vimeo_id` OU `youtube_url` — não ambos ao mesmo tempo (radio ou tabs simples: "Vimeo" / "YouTube")

**Player de vídeo — componente existente (provavelmente `VimeoPlayer.tsx` ou similar):**

- Detectar tipo de vídeo: se `lesson.youtube_url` preenchido → usar YouTube iframe; senão → usar Vimeo
- YouTube player via iframe API padrão: `https://www.youtube.com/embed/{VIDEO_ID}?enablejsapi=1`
- Thumbnail customizado: se `video_thumbnail_url` preenchido, exibir como poster antes do play
- Auto-resume: ao carregar o player, verificar `lesson_progress.resume_position_secs` e setar `startSeconds` no embed (YouTube: `&start={secs}`, Vimeo: `#t={secs}s`)
- Salvar position: ao pausar ou a cada 30 segundos, fazer PATCH em `lesson_progress.resume_position_secs`

**Completion requirement:**

- Ao concluir o vídeo OU ao atingir `completion_percent`% do total, marcar aula como concluída via PATCH existente em `lesson_progress`
- Se `completion_percent = 0`: comportamento atual (marcar ao clicar em "Concluir")

**API `PATCH /api/progress/[lessonId]` ou equivalente:**

- Aceitar `resume_position_secs` no body
- Atualizar `lesson_progress.resume_position_secs`

### OUT

- Upload de vídeo direto (self-hosted) — necessita bucket + CDN, fora de escopo
- YouTube playlist ou canal inteiro como aula
- Vídeo protegido (sem embedding) — bypass de restrições do YouTube
- Player customizado com controles próprios (usar iframe padrão do YouTube/Vimeo)
- Relatório de tempo assistido por aluno no admin

## Technical Notes

### Extração do YouTube ID

```typescript
function extractYouTubeId(url: string): string | null {
  const patterns = [
    /youtube\.com\/watch\?v=([^&\n?#]+)/,
    /youtu\.be\/([^&\n?#]+)/,
    /youtube\.com\/embed\/([^&\n?#]+)/,
  ]
  for (const pattern of patterns) {
    const match = url.match(pattern)
    if (match) return match[1]
  }
  return null
}
```

### Auto-resume — salvar posição

```typescript
// A cada 30s e no onPause, fazer:
async function savePosition(lessonId: string, positionSecs: number) {
  await fetch(`/api/progress/${lessonId}`, {
    method: 'PATCH',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ resume_position_secs: positionSecs }),
  })
}
```

### Completion requirement

```typescript
// Ao receber evento de progresso do player:
function onVideoProgress(currentSecs: number, totalSecs: number, lesson: LessonRow) {
  if (lesson.completion_percent > 0) {
    const watchedPercent = (currentSecs / totalSecs) * 100
    if (watchedPercent >= lesson.completion_percent) {
      markLessonComplete(lesson.id) // PATCH lesson_progress.completed = true
    }
  }
}
```
