## Story

**As a** visitante ou aluno da plataforma,
**I want** poder assinar um plano com billing semestral ao escolher a opção "Semestral" na página de planos,
**so that** o checkout Stripe seja criado com o price correto de 6 meses e minha subscription reflita esse período.

## Business Value

Sem o checkout funcional, a opção semestral visível (Story 19.2) não tem ação — gera frustração ao visitante que clica em "Assinar" e recebe uma cobrança mensal. A integração com Stripe Price de 6 meses (`interval_count: 6`) completa o ciclo do Epic 19 e efetivamente monetiza a nova opção de billing.

## Complexity Estimate

**T-shirt:** S (Small) — ~2-3h
**Story Points:** 2

## Scope

### IN

- Suporte ao parâmetro `billingPeriod: 'semiannual'` no provider/função de criação de Stripe Checkout Session
- Criar ou referenciar Stripe Price com `recurring.interval: 'month', recurring.interval_count: 6` para cada plano que tiver `stripe_price_id_semiannual` configurado
- Armazenar `stripe_price_id_semiannual` no banco (coluna já criada em 19.1; valor preenchido pelo admin ou via seed/script)
- Webhook: o handler existente de `checkout.session.completed` deve registrar `billing_period: 'semiannual'` quando o price usado for o semestral

### OUT

- UI do botão de checkout semestral (está na Story 19.2 — botão "Assinar" redireciona com `billingPeriod: 'semiannual'`)
- Criação manual de Prices no Stripe Dashboard (instrução para o admin, fora do código)
- Tela de gerenciamento de billing

## Dependencies

- **Story 19.1 deve estar completa** — coluna `stripe_price_id_semiannual` deve existir
- **Story 19.2 deve estar completa** — UI envia `billingPeriod: 'semiannual'` ao clicar em assinar
- Stripe Checkout Session provider existente (verificar localização: `src/app/api/checkout/` ou similar — criado no Epic 16)
- Webhook de `checkout.session.completed` existente (Epic 16)

## Acceptance Criteria

- **AC-1:** Ao chamar o provider de checkout com `billingPeriod: 'semiannual'`, a Stripe Session é criada com o `price_id` correspondente ao `stripe_price_id_semiannual` do plano selecionado
- **AC-2:** O Stripe Price referenciado possui `recurring.interval: 'month'` e `recurring.interval_count: 6` (verificável no Stripe Dashboard ou via `stripe.prices.retrieve`)
- **AC-3:** Se `stripe_price_id_semiannual` for `null` ou vazio para o plano selecionado, o checkout retorna erro `400` com mensagem clara ("Plano semestral não disponível para este plano")
- **AC-4:** O webhook `checkout.session.completed` ao processar uma subscription semestral registra `billing_period: 'semiannual'` na tabela `subscriptions`
- **AC-5:** O fluxo de checkout mensal e anual continua funcionando sem regressão após as mudanças

## Technical Notes

### Arquivo a modificar

Verificar onde o Stripe Checkout Session é criado — provavelmente `src/app/api/checkout/route.ts` ou `src/app/api/checkout/session/route.ts` (criado no Epic 16).

### Lógica de seleção de price_id

```typescript
const getPriceId = (plan: Plan, billingPeriod: BillingPeriod): string => {
  if (billingPeriod === 'monthly') return plan.stripe_price_id_monthly
  if (billingPeriod === 'semiannual') {
    if (!plan.stripe_price_id_semiannual) {
      throw new Error('Plano semestral não disponível para este plano')
    }
    return plan.stripe_price_id_semiannual
  }
  return plan.stripe_price_id_annual
}
```

### Como criar Stripe Price semestral

O Stripe não tem conceito nativo de "semestral" — usa `interval: 'month'` com `interval_count: 6`:

```typescript
// Criar via Stripe SDK (executar uma vez por plano, guardar ID no banco)
const price = await stripe.prices.create({
  currency: 'brl',
  unit_amount: plan.price_semiannual, // em centavos
  recurring: {
    interval: 'month',
    interval_count: 6,
  },
  product: plan.stripe_product_id, // produto existente no Stripe
})
// Salvar price.id em plans.stripe_price_id_semiannual
```

[AUTO-DECISION] A criação dos Prices no Stripe é responsabilidade do admin (via Stripe Dashboard) ou de um script one-off. Esta story implementa apenas o uso do `price_id` já configurado no banco. O dev deve documentar as instruções para o admin criar o Price semestral no Stripe Dashboard.

### Webhook: identificar billing_period pelo price_id

```typescript
// Em checkout.session.completed handler (já existente no Epic 16)
// Adicionar lógica de detecção do billing_period:
const priceId = session.line_items?.data[0]?.price?.id

const { data: plan } = await supabase
  .from('plans')
  .select('id, stripe_price_id_monthly, stripe_price_id_annual, stripe_price_id_semiannual')
  .or(
    [
      `stripe_price_id_monthly.eq.${priceId}`,
      `stripe_price_id_annual.eq.${priceId}`,
      `stripe_price_id_semiannual.eq.${priceId}`,
    ].join(',')
  )
  .single()

const billingPeriod =
  priceId === plan?.stripe_price_id_semiannual
    ? 'semiannual'
    : priceId === plan?.stripe_price_id_annual
      ? 'annual'
      : 'monthly'
```
