# Story 26.1: Admin UX — Fundação shadcn/ui + 1ª migração (detalhe do aluno)

## Story

**As a** Fábio Borges (administrador),
**I want** os componentes do admin construídos com shadcn/ui, limpos e consistentes,
**so that** o painel fique mais objetivo e fácil de manejar.

## Business Value

O projeto tinha shadcn inicializado mas ~10% montado (só `button`, tema ausente, 40 arquivos com `<style>` inline). Esta story monta a fundação (tema com a marca) e migra a primeira tela como padrão/vitrine, destravando a migração incremental do resto.

## Complexity Estimate

**T-shirt:** M — ~4h (fase 1 de N) · **Story Points:** 5

## Scope

### IN

- **Fundação:** tokens de tema shadcn (light+dark) no `globals.css` com `primary` = azul da marca; instalação dos componentes (card, input, label, select, badge, table, separator, tabs, dropdown-menu, dialog). Estilo `base-nova` (@base-ui/react).
- **1ª migração:** `AdminStudentDetail` reescrito com Card/Button/Badge + Tailwind (removido `<style>` inline).

### OUT

- Migração das demais telas do admin (lista, dashboard, cursos, planos) — próximas fases.
- Telas públicas/checkout/player.

## Acceptance Criteria

1. Tokens shadcn definidos; build passa (Tailwind resolve `bg-primary`, `text-muted-foreground`, etc.).
2. Detalhe do aluno renderiza com componentes shadcn, funções intactas (matricular/desmatricular/plano/progresso).
3. Sem regressão: coexiste com o CSS custom das telas ainda não migradas.

## Dependencies

## Definition of Done

## Dev Notes

- Tema shadcn adicionado como bloco `:root`/`.dark` separado no `globals.css`, sem remover as vars custom (`--ink`, `--blue`…) que as 40 telas legadas ainda usam. Migração incremental sem big-bang.
- `primary` = `211 86% 55%` (azul da Mentoria). Estilo `base-nova` → componentes usam `@base-ui/react` (já instalado; sem radix).
- Select nativo estilizado com classes shadcn (evita risco da API base-ui do Select nesta 1ª iteração).
