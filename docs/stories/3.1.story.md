## Story

**As a** aluno autenticado com acesso liberado a uma aula,
**I want** assistir as aulas em vídeo com controle de progresso automático,
**so that** meu progresso seja salvo continuamente, eu saiba exatamente onde parei, e o curso seja marcado como concluído ao atingir 90% do conteúdo assistido.

## Business Value

O player de vídeo é o coração da plataforma — sem ele, o aluno não tem acesso ao conteúdo pelo qual pagou. Esta story entrega a experiência central de consumo do produto: assistir aulas com progresso persistido. O auto-save elimina a frustração de perder o ponto de parada. A marcação automática de conclusão por aula (90%) é o gatilho que desbloqueará certificados (Sprint 5) e permitirá ao admin ver taxa de conclusão (Sprint 6).

## Scope

**IN (o que esta story entrega):**

- Componente `<VimeoPlayer>` com embed via iframe da API do Vimeo (sem oEmbed — embed direto por `vimeo_id`)
- Integração do design exportado do Claude Design (HTML/CSS/JS → componente React/Tailwind)

- Hook `useProgress(courseId)` — lê e sincroniza progresso via TanStack Query

## Acceptance Criteria

1. **AC1 — Player renderiza:** Dado um `vimeo_id` válido na aula, quando o aluno acessa `/dashboard/curso/[id]`, o componente `<VimeoPlayer>` renderiza o vídeo via iframe Vimeo embed (`https://player.vimeo.com/video/{vimeo_id}`) sem erros de console.

2. **AC2 — Auto-save de progresso:** Dado que o aluno está assistindo um vídeo, quando passam 30 segundos de reprodução, o cliente chama `POST /api/progress` com `{ lessonId, watchedSecs }` e a API faz upsert na tabela `lesson_progress` retornando `{ updated: true }`.

3. **AC3 — Marcação de conclusão automática:** Dado que o aluno assistiu ≥ 90% da duração da aula (`watched_secs >= duration_secs * 0.9`), quando o próximo auto-save ocorre, a API persiste `completed: true` na `lesson_progress` daquela aula.

4. **AC4 — Progresso persiste entre sessões:** Dado que o aluno já assistiu parte de uma aula, quando ele retorna à página do curso, o player inicia no ponto onde parou (`watched_secs` da última `lesson_progress`).

5. **AC5 — Barra de progresso do curso:** Dado que o aluno tem aulas marcadas como `completed: true`, quando visualiza a página do curso, a barra de progresso exibe a porcentagem correta `(aulas concluídas / total aulas do curso) × 100`.

6. **AC6 — Lista de aulas com estado visual:** Dado que o aluno visualiza a lista de módulos/aulas, quando há progresso registrado, aulas concluídas exibem ícone/indicador visual distinto (ex: checkmark), aulas em progresso exibem indicador diferente, e aulas não iniciadas exibem estado neutro.

7. **AC7 — Design integrado:** O layout da página `/dashboard/curso/[id]` corresponde ao design exportado do Claude Design, adaptado para React/Tailwind com tokens do projeto.

8. **AC8 — Validação da API:** A rota `POST /api/progress` valida o body com Zod (`lessonId: string uuid`, `watchedSecs: number >= 0`, `completed?: boolean`) e retorna `400` com mensagem de erro se inválido.

9. **AC9 — Segurança:** A API `POST /api/progress` verifica autenticação via Supabase session server-side. Usuário não autenticado recebe `401`. RLS da tabela `lesson_progress` impede que um usuário salve progresso de outro usuário.

10. **AC10 — Typecheck e lint passam:** `npm run typecheck` e `npm run lint` executam sem erros após a implementação.
