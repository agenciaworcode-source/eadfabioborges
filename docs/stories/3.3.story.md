## Story

**As a** aluno autenticado com acesso a uma aula,
**I want** enviar minha tarefa prática (PDF, imagem ou ZIP) diretamente na plataforma,
**so that** o instrutor possa corrigir minha entrega e eu receba nota e feedback sem sair da plataforma.

## Business Value

As tarefas práticas são o diferencial pedagógico da mentoria — permitem que o aluno aplique o conteúdo aprendido e receba feedback personalizado do Fábio. Sem upload integrado, o aluno precisa enviar por WhatsApp ou email, gerando caos operacional para o mentor. Esta story fecha o loop de aprendizado da Sprint 4: assistir → avaliar (quiz) → praticar (tarefa). A página admin de correção entrega ao Fábio a ferramenta operacional mais crítica do dia a dia.

## Complexity Estimate

**Story Points:** 3

## Scope

**IN (o que esta story entrega):**

- Componente `<AssignmentUpload>` — drag-and-drop com fallback click, aceita PDF/imagem/ZIP, máx 50 MB, preview do arquivo selecionado
- `POST /api/assignment/upload` — recebe FormData, valida tipo e tamanho (Zod), faz upload para Supabase Storage bucket `submissions`, insere registro em `submissions`
- `GET /api/assignment/[assignmentId]` — retorna dados da assignment + submission do próprio aluno (ou null se não enviou)
- Hook `useAssignment(lessonId)` — busca assignment da aula + submission do aluno via TanStack Query
- Integração no `CoursePlayerLayout` — exibir `<AssignmentUpload>` abaixo do quiz/player quando a aula tem tarefa vinculada
- Layout admin básico `src/app/(admin)/layout.tsx` — sidebar admin + proteção de rota `role = admin`
- Página admin `src/app/(admin)/admin/tarefas/[id]/page.tsx` — lista todas as submissions de uma assignment + formulário de correção (nota 0-100 + feedback)
- API `PATCH /api/assignment/submission/[submissionId]` — admin atualiza `grade` + `feedback` + `graded_at`

**OUT (o que esta story NÃO entrega):**

- Email de notificação ao aluno sobre correção (→ Sprint 6)
- Download em lote de submissions (→ Sprint 6 admin)
- Painel admin completo (alunos, cursos, relatórios → Sprint 6)
- Antivirus/scan de arquivo enviado (→ fora do escopo do projeto)

## Acceptance Criteria

1. **AC1 — AssignmentUpload renderiza para aula com tarefa:** Dado que a aula possui uma assignment vinculada (`assignment.lesson_id = lesson.id`), quando o aluno visualiza a página do curso, o componente `<AssignmentUpload>` é exibido com título e instruções da tarefa.

2. **AC2 — Upload de arquivo válido:** Dado que o aluno arrasta ou seleciona um arquivo PDF/imagem/ZIP (≤ 50 MB), quando clica em "Enviar", o arquivo é enviado para `POST /api/assignment/upload` e a API retorna `{ submissionId, fileUrl }`.

3. **AC3 — Validação de tipo e tamanho:** A API rejeita arquivos com tipo inválido (não PDF/imagem/ZIP) com `400 { error: 'Tipo de arquivo não permitido' }`. Arquivos acima de 50 MB retornam `400 { error: 'Arquivo muito grande (máx 50 MB)' }`. A validação é feita também no cliente antes do envio.

4. **AC4 — Submission persistida:** Após upload bem-sucedido, a API insere em `submissions` com `{ user_id, assignment_id, file_url }` e o componente exibe o estado "Entregue" com link para o arquivo.

5. **AC5 — Reenvio bloqueado após entrega:** Uma vez que o aluno entregou a tarefa, o componente exibe o estado "Tarefa enviada" (sem novo botão de upload). O aluno não pode reenviar pela interface.

6. **AC6 — Persistência entre sessões:** Quando o aluno retorna à página, o componente carrega o estado correto via `useAssignment` — se já entregou, mostra "Enviada"; se não entregou, mostra o formulário.

7. **AC7 — Página admin lista submissions:** Dado que o admin acessa `/admin/tarefas/[id]`, a página exibe a lista de todas as submissions daquela assignment com: nome do aluno, data de envio, link para o arquivo, status (corrigida / aguardando).

8. **AC8 — Formulário de correção:** O admin pode selecionar uma submission e preencher nota (0-100) + feedback em texto. Ao salvar, `PATCH /api/assignment/submission/[submissionId]` atualiza `grade`, `feedback` e `graded_at`.

9. **AC9 — Proteção da rota admin:** A página `/admin/tarefas/[id]` só é acessível por usuários com `role = admin`. O middleware já redireciona não-admins para `/dashboard`. A API `PATCH` verifica `role = admin` server-side e retorna `403` se não for admin.

10. **AC10 — Segurança do upload:** A API verifica autenticação via Supabase session. Usuário não autenticado recebe `401`. Cada arquivo é armazenado em path isolado por usuário: `submissions/{user_id}/{submissionId}/{filename}`.

11. **AC11 — Typecheck e lint passam:** `npm run typecheck` e `npm run lint` executam sem erros após a implementação.

```
  Body: FormData { assignmentId: string; file: File (max 50MB) }
  Auth: Requer sessão

  Validação: tipo (PDF/img/ZIP), tamanho máx 50MB
```

### Supabase Storage

- Bucket: `submissions` (já criado na Story 1.2 ou criar via SQL migration se não existir)
- Path de upload: `{user_id}/{crypto.randomUUID()}/{originalFilename}`
- Método: `supabase.storage.from('submissions').upload(path, file)`
- URL pública: `supabase.storage.from('submissions').getPublicUrl(path).data.publicUrl`
- **ATENÇÃO:** O bucket `submissions` precisa ter política de acesso: upload autenticado, leitura pública (ou somente autenticado)
