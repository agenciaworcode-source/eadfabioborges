## Story

**As a** visitante ou aluno da plataforma EAD Mentoria Fábio Borges,
**I want** poder me cadastrar com e-mail/senha ou entrar com minha conta Google, e ter minhas rotas privadas protegidas automaticamente,
**so that** só eu tenha acesso ao meu dashboard e conteúdo, com experiência de login fluida e segura.

## Business Value

Sem autenticação, nenhuma área protegida existe — o produto inteiro fica inacessível. Esta story é o **portão de entrada** de toda a plataforma: sem ela, o Fábio não consegue vender acesso a nada. Com ela, qualquer visitante pode criar conta ou logar com Google em segundos, e o middleware garante que nenhuma rota privada vaze conteúdo para usuários não autenticados.

## Acceptance Criteria

1. `POST /auth/cadastro` — formulário com campos: nome completo, e-mail, senha (mín. 8 chars) + confirmação senha; Zod validação client-side; erro inline se campos inválidos
2. `POST /auth/login` — campos: e-mail + senha; ao autenticar com sucesso, redirecionar para `returnUrl` ou `/dashboard`
3. Botão "Entrar com Google" em `/auth/login` e `/auth/cadastro` chama `supabase.auth.signInWithOAuth({ provider: 'google' })`
4. `src/app/auth/callback/route.ts` — handler GET que troca code por sessão via `supabase.auth.exchangeCodeForSession(code)` e redireciona para `/dashboard`
5. `middleware.ts` — matcher: `['/dashboard/:path*', '/admin/:path*']`; rota sem sessão → `redirect('/auth/login?returnUrl=...')`; admin sem role admin → `redirect('/dashboard')`
6. Hook `useAuth` exporta `user`, `session`, `loading`, `signOut()` usando o browser client do Supabase
7. Zustand store `useAuthStore` com `user`, `setUser`
8. `/auth/recuperar-senha` — campo e-mail + botão "Enviar link"; chama `supabase.auth.resetPasswordForEmail(email)`; mostra mensagem de sucesso após envio
9. Testes unitários: `tests/unit/hooks/use-auth.test.ts` com mock do `createBrowserClient`

### Middleware — Implementação de Referência

[Source: docs/architecture/10-arquitetura-frontend.md — 10.4 Middleware (Auth Guard)]

```typescript
// middleware.ts
import { createServerClient } from '@supabase/ssr'
import { NextResponse } from 'next/server'

export async function middleware(request: NextRequest) {
  const supabase = createServerClient(/* cookies */)
  const {
    data: { user },
  } = await supabase.auth.getUser()

  const isDashboard = request.nextUrl.pathname.startsWith('/dashboard')
  const isAdmin = request.nextUrl.pathname.startsWith('/admin')

  if ((isDashboard || isAdmin) && !user) {
    const loginUrl = new URL('/auth/login', request.url)
    loginUrl.searchParams.set('returnUrl', request.nextUrl.pathname)
    return NextResponse.redirect(loginUrl)
  }

  if (isAdmin) {
    const { data: profile } = await supabase
      .from('users')
      .select('role')
      .eq('id', user!.id)
      .single()
    if (profile?.role !== 'admin') {
      return NextResponse.redirect(new URL('/dashboard', request.url))
    }
  }
  return NextResponse.next()
}

export const config = {
  matcher: ['/dashboard/:path*', '/admin/:path*'],
}
```
