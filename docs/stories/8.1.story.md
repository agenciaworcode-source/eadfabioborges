## Story

**As a** aluno ou administrador,
**I want** receber emails automáticos nos momentos-chave da minha jornada (boas-vindas, certificado emitido, tarefa corrigida e lembrete de inatividade),
**so that** eu me mantenha engajado com a plataforma e saiba imediatamente quando algo importante aconteceu na minha conta.

## Business Value

Emails transacionais são o canal de retenção mais eficiente da plataforma. Sem eles: o aluno novo não tem confirmação de cadastro; a emissão do certificado passa despercebida; o feedback do mentor demora a chegar; alunos somem sem aviso. Integrar o Resend fecha o loop de comunicação da Sprint 6 (PRD seção 7), sem depender dos pagamentos ainda deferidos para Sprint 3.

- **Hook tarefa corrigida** em `src/app/api/assignment/submission/[submissionId]/route.ts` — dispara email após `UPDATE` de nota/feedback; busca `email` do aluno via join em `users`

- **`.env.example`** — adicionar `CRON_SECRET=` (já tem `RESEND_API_KEY`)
- Todos os disparos de email são **fire-and-forget** (não bloqueiam a resposta HTTP) via `void sendEmail(...)`
- Erros de envio são apenas logados com `console.error` — nunca retornam 500 para o cliente

### Cliente Resend singleton

```typescript
// src/lib/resend.ts
import { Resend } from 'resend'
import type { ReactElement } from 'react'

const resend = process.env.RESEND_API_KEY ? new Resend(process.env.RESEND_API_KEY) : null

export async function sendEmail(to: string, subject: string, react: ReactElement): Promise<void> {
  if (!resend) {
    console.warn('[resend] RESEND_API_KEY not set — email not sent:', subject)
    return
  }
  const { error } = await resend.emails.send({
    from: 'Mentoria Fábio Borges <noreply@ead.fabioborgesoficial.com.br>',
    to,
    subject,
    react,
  })
  if (error) console.error('[resend] send error:', error)
}
```

### Cron endpoint — inatividade

```typescript
// src/app/api/cron/inactivity-reminder/route.ts
import { NextResponse } from 'next/server'
import { createServiceClient } from '@/lib/supabase/service' // service role, para auth.users
```

> **Nota:** `createServiceClient()` usa `SUPABASE_SERVICE_ROLE_KEY` (já em `.env.example`) para chamar `supabase.auth.admin.listUsers()`. O client atual `createClient()` usa o cookie-based client — não serve para cron jobs sem sessão.

Criar `src/lib/supabase/service.ts`:

```typescript
import { createClient } from '@supabase/supabase-js'
export function createServiceClient() {
  return createClient(process.env.NEXT_PUBLIC_SUPABASE_URL!, process.env.SUPABASE_SERVICE_ROLE_KEY!)
}
```

Lógica do cron:

```typescript
export async function GET(request: Request) {
  const secret = request.headers.get('authorization')?.replace('Bearer ', '')
  if (secret !== process.env.CRON_SECRET) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
  }

  const supabase = createServiceClient()
  const now = new Date()
  const THRESHOLDS = [7, 14, 30]
  let sent = 0

  for (const days of THRESHOLDS) {
    const from = new Date(now.getTime() - (days + 1) * 86400_000)
    const to = new Date(now.getTime() - days * 86400_000)

    // Buscar em tabela users (last_seen ou last_sign_in_at via auth join)
    const { data: users } = await supabase
      .from('users')
      .select('id, name, email, last_seen_at')
      .gte('last_seen_at', from.toISOString())
      .lt('last_seen_at', to.toISOString())

    for (const u of users ?? []) {
      if (!u.email) continue
      void sendEmail(
        u.email,
        `Faz ${days} dias que você não acessa a Mentoria`,
        <InativoLembreteEmail name={u.name ?? 'Aluno'} days={days} dashboardUrl={`${process.env.NEXT_PUBLIC_APP_URL}/dashboard`} />
      )
      sent++
    }
  }

  return NextResponse.json({ sent })
}
```

> **Nota sobre `last_seen_at`:** verificar se a tabela `users` tem coluna `last_seen_at`. Se não existir, usar `updated_at` ou `last_sign_in_at` via `auth.users` com `supabase.auth.admin.listUsers()` com paginação. Documentar decisão no Change Log.

### Padrão fire-and-forget

Todos os `sendEmail(...)` são chamados com `void` — nunca com `await`. Isso garante que falha de email não retorna 500 nem atrasa resposta HTTP.

```typescript
// CORRETO:
void sendEmail(to, subject, <Template />)
return NextResponse.json({ updated: true })

// ERRADO:
await sendEmail(to, subject, <Template />)  // bloqueia resposta
```
