## Story

**As a** desenvolvedor do projeto EAD Mentoria Fábio Borges,
**I want** o Supabase configurado localmente e na cloud com todas as migrations, RLS policies e tipos TypeScript gerados,
**so that** as stories seguintes (Auth, Pagamentos, Conteúdo) possam consumir o banco com segurança e tipagem completa desde o início.

## Business Value

O banco de dados é a fundação de toda a lógica de negócio: matrícula, controle de acesso por plano, progresso de aluno, certificados. Sem as migrations e RLS corretas, qualquer feature posterior requer retrabalho — e pior, pode expor dados de um aluno para outro. Esta story garante que o banco está correto, seguro e documentado antes de qualquer linha de código de produto.

### Clientes Supabase (padrão @supabase/ssr)

[Source: docs/architecture/11-arquitetura-backend.md]

```typescript
// src/lib/supabase/client.ts — uso em Client Components
import { createBrowserClient } from '@supabase/ssr'
import type { Database } from '@/types/database'

export function createClient() {
  return createBrowserClient<Database>(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
  )
}

// src/lib/supabase/server.ts — uso em Server Components e Route Handlers
import { createServerClient } from '@supabase/ssr'
import { cookies } from 'next/headers'
import type { Database } from '@/types/database'

export function createClient() {
  const cookieStore = cookies()
  return createServerClient<Database>(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll() {
          return cookieStore.getAll()
        },
        setAll(cookiesToSet) {
          cookiesToSet.forEach(({ name, value, options }) => cookieStore.set(name, value, options))
        },
      },
    }
  )
}
```
