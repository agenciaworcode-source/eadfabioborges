## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** ver o progresso detalhado de cada aluno por curso na área admin, com drill-down por módulo,
**so that** eu possa identificar rapidamente quais alunos estão engajados, quais estão travados e quais módulos têm maior abandono.

## Business Value

O painel de progresso de alunos transforma dados de consumo em inteligência de negócio: Fábio pode identificar alunos que precisam de suporte antes do churn, descobrir módulos problemáticos e tomar decisões de curadoria de conteúdo com base em dados reais de conclusão. Sem essa visibilidade, a gestão da plataforma é cega.

## Complexity Estimate

**T-shirt:** M (Medium) — ~4-5h
**Story Points:** 3

## Scope

### IN

- Na área admin, na tela de detalhes do aluno (`/admin/alunos/[userId]` — existente ou nova):
  - Adicionar seção/aba "Progresso por Curso"
  - Listar cada curso matriculado com:
    - Nome do curso
    - Progress bar visual (0-100%)
    - Percentual concluído (ex: "67% concluído")
    - Data de última atividade (ex: "Última atividade: 15/06/2026" ou "Nunca acessou")
    - Data de matrícula e data de expiração (se houver)
  - Expandir/colapsar módulos de cada curso:
    - Nome do módulo
    - Número de aulas completadas / total (ex: "3 de 5 aulas")
    - Lista de aulas com ícone ✓ (completada) ou ○ (pendente)
- Estado de loading durante busca do endpoint
- Mensagem de estado vazio: "Aluno não possui matrículas" se não houver cursos

### OUT

- Filtros ou busca dentro do progresso
- Exportação CSV/PDF do progresso
- Anotações ou notas do admin sobre o aluno
- Envio de email ao aluno diretamente da tela
- Gráfico de linha de progresso ao longo do tempo

## Dependencies

- **Story 21.1 deve estar completa** — endpoint `GET /api/admin/users/[userId]/progress` deve existir
- Tela de detalhes do aluno em `/admin/alunos/[userId]` — verificar se existe e sua estrutura atual
- Componentes: `Progress`, `Collapsible`/`Accordion`, `Badge` do shadcn/ui
- Acesso à área admin protegido (middleware existente)

## Acceptance Criteria

- **AC-1:** Em `/admin/alunos/[userId]`, há uma seção visível "Progresso por Curso" com a lista de cursos matriculados
- **AC-2:** Cada curso exibe: nome do curso, progress bar (0-100%), percentual textual, data de última atividade, data de matrícula
- **AC-3:** A progress bar reflete o `progress_percent` retornado pelo endpoint (ex: 67% → barra 67% preenchida)
- **AC-4:** Clicar em um curso expande a lista de módulos; cada módulo mostra "X de Y aulas concluídas"
- **AC-5:** Expandir um módulo exibe a lista de aulas com indicador visual de concluída (✓) ou pendente (○)
- **AC-6:** Data de última atividade exibe "Nunca acessou" quando `last_activity_at` for `null`
- **AC-7:** Durante carregamento do endpoint, um skeleton/spinner é exibido no lugar da lista
- **AC-8:** Se o aluno não tem matrículas, exibe mensagem "Aluno não possui matrículas ativas"

### Estrutura de dados esperada do endpoint (Story 21.1)

```typescript
type CourseProgress = {
  course_id: string
  course_title: string
  enrolled_at: string
  expires_at: string | null
  total_lessons: number
  completed_lessons: number
  progress_percent: number // 0-100
  last_activity_at: string | null // ISO timestamp ou null
  modules: {
    module_id: string
    module_title: string
    module_order: number
    total_lessons: number
    completed_lessons: number
    progress_percent: number
    lessons: {
      lesson_id: string
      lesson_title: string
      is_completed: boolean
    }[]
  }[]
}

type ProgressResponse = CourseProgress[]
```
