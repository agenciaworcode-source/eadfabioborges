## Story

**As a** Fábio Borges (administrador da plataforma),
**I want** que a plataforma passe por um checklist completo de segurança, performance e QA antes de ir ao ar,
**so that** o go-live aconteça com zero surpresas: todos os fluxos críticos testados, headers de segurança configurados, backups ativos e monitoramento funcionando.

## Business Value

A plataforma tem todas as funcionalidades implementadas. O go-live sem hardening expõe dados de alunos, abre brechas de segurança e arrisca falhas em produção sem alertas. Esta story é a última barreira antes do acesso real de alunos pagantes — e é também a última oportunidade de validar os fluxos end-to-end de ponta a ponta.

## Scope

### IN

- **Validação Zod em todas as API routes** — verificar que TODAS as routes têm validação de input; adicionar onde falta (prioridade: rotas que recebem dados de usuário)
- **Headers de segurança no `next.config.mjs`** — adicionar: `X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY`, `Referrer-Policy: strict-origin-when-cross-origin`, `Permissions-Policy`, `X-XSS-Protection: 1; mode=block`
- **Rate limiting básico** — adicionar rate limit por IP em `/api/auth/*`, `/api/checkout/*` e `/api/webhooks/*` usando `src/middleware.ts` (já existe) com contagem em memória ou via Vercel middleware headers
- **Smoke test checklist** — arquivo `docs/go-live/smoke-test-checklist.md` com todos os fluxos críticos para teste manual antes do deploy
- **Backups** — documentar em `docs/go-live/backup-setup.md` como configurar backup automático do Supabase (point-in-time recovery já habilitado no plano Pro; criar schedule no Supabase Dashboard)
- **Monitoramento** — documentar em `docs/go-live/monitoring-setup.md`: configurar UptimeRobot para `/api/health` + alerta WhatsApp; criar `src/app/api/health/route.ts` (GET, retorna `{ status: 'ok', ts: ISO }`)
- **Variáveis de produção** — `docs/go-live/env-production-checklist.md` com lista de TODAS as variáveis de ambiente necessárias em produção (Supabase, Stripe, Resend, etc.)
- **`npm run build` sem erros** — verificar que o build de produção passa limpo

## Technical Notes

### Headers de segurança — next.config.mjs

```javascript
// next.config.mjs
const securityHeaders = [
  { key: 'X-Content-Type-Options', value: 'nosniff' },
  { key: 'X-Frame-Options', value: 'DENY' },
  { key: 'X-XSS-Protection', value: '1; mode=block' },
  { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },
  { key: 'Permissions-Policy', value: 'camera=(), microphone=(), geolocation=()' },
]

const nextConfig = {
  // ... config existente ...
  async headers() {
    return [
      {
        source: '/(.*)',
        headers: securityHeaders,
      },
    ]
  },
}

export default nextConfig
```

### Fluxos críticos para smoke-test-checklist.md

1. **Fluxo 1 — Cadastro novo aluno:** visita `/auth/cadastro` → preenche form → confirma email → loga → vê dashboard
2. **Fluxo 2 — Login com Google:** visita `/auth/login` → "Entrar com Google" → callback → dashboard
3. **Fluxo 3 — Catálogo público:** visita `/cursos` sem login → vê cursos → clica no curso → vê detalhe → clica "Quero me matricular" → vai para login
4. **Fluxo 4 — Checkout curso:** aluno logado em `/cursos/[slug]` → clica "Quero me matricular" → Stripe Checkout → paga em sandbox → `/checkout/sucesso` → enrollment criado → acessa aula
5. **Fluxo 5 — Player + progresso:** aluno matriculado → acessa aula → assiste vídeo → progresso salvo → marca como concluída
6. **Fluxo 6 — Quiz:** aluno na aula com quiz → responde quiz → vê resultado → acesso à próxima aula
7. **Fluxo 7 — Certificado:** aluno conclui curso → PDF gerado → recebe email → página `/certificado/[uuid]` verificável
8. **Fluxo 8 — Admin matrícula manual:** admin loga → `/admin/alunos` → "Matricular manualmente" → busca email → seleciona curso → confirma → enrollment criado
9. **Fluxo 9 — Admin cursos:** admin → `/admin/cursos` → cria curso → adiciona módulo → adiciona aula (vimeo_id) → publica
10. **Fluxo 10 — Planos:** visita `/planos` → vê 4 planos → clica "Assinar plano" → Stripe Subscription Checkout
